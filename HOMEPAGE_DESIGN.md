# 首页现代化设计说明

## 设计概述

根据仓库指令和现代化设计趋势，Rune Docs 首页全面转向面向开发者的 Bento Grid (便当盒)与深色模式沉浸式设计。新设计受顶级开发者工具 (Vercel, Supabase) 启发，结合代码实时展示、生态长廊与聚合卡片，提供专业、高信息密度的文档索引体验。

## 页面结构

首页采用六层大区级架构：

1. **Hero 区块** - 超大尺寸排版，辅以模拟终端命令行（展示 SDK 安装与部署），直接体现"Developer First"氛围。
2. **Capabilities (核心能力) Bento Grid** - 摒弃对称的三等分均卡。大型区块展示 Rune主平台（包括光环悬停效果）；右侧堆叠魔哈网格和 Boss 平台。
3. **Code Showcase (代码呈现)** - 对标现代 API 工具，高亮渲染真实 TypeScript 调用代码，体现便捷的 SDK 开发体验。
4. **Ecosystem (生态跑马灯)** - 以无缝循环的 Framer Motion 滚动长廊展示 PyTorch, vLLM 等框架，增强信任。
5. **Resources (生态分类资源)** - 提供快捷的深层文档直达入口，包含透明光效 hover 网格卡片。
6. **CTA (行动号召)** - 品牌主色带圆角沉浸式底部 Banner。

## 设计亮点

### 沉浸式暗色体验 (`bgcolor: #000000`)
- 通过覆盖 `Box` 组件级的 `bgcolor` 将整个首页的视觉底色压制为纯黑 `#000000` 或 `#0a0a0a`。
- 使用 `alpha()` 函数实现大颗粒的 Radial Gradient 径向渐变，制造局部的高级「光晕 (Glow)」现象。

### Hero 模拟终端
- 全新绘制 Mac 风格的深色终端窗体（模拟安装 `@xiaoshiai/rune-sdk` 等命令），通过闪烁的光标动画建立工程师第一印象觉。

### Asymmetric Bento Grid
- 使用 `display: 'grid'` 和 `gridTemplateColumns` 对三大产品线进行差异化空间分配。Rune 主平台采用 `span 2`，体现主入口地位；另外两平台使用 `span 1`。

### Framer Motion 高级交互
- 所有区块依赖 `MotionViewport` 搭配 `varFade('inUp')` / `varFade('inDown')` 确保滚动连贯感。
- **生态滚动长廊**: 利用 `animate={{ x: [0, -1500] }}` 实现在 `x` 轴上无穷平移（`Infinity`）。

## 颜色方案

为了配合深色底图，颜色的透明度进行了大面积运用（例如 `alpha('#1877F2', 0.1)`），但品牌的本源颜色保留如下核心定义：

| 产品/功能 | 主色 | 应用场景 |
|----------|------|---------|
| Rune 智算平台 | #1877F2 | 图标、边框、按钮 |
| 魔哈仓库 | #00A76F | 图标、边框、按钮 |
| Boss 运营平台 | #7635DC | 图标、边框、按钮 |

- `cta.tsx` - 行动号召

## 后续优化建议

1. **内容完善**：根据需要可以恢复其他区块（能力总览、快速上手等）
2. **图片优化**：考虑为产品卡片添加插图或截图
3. **数据动态化**：产品描述和链接可接入配置文件
4. **国际化**：确保所有文案都通过 i18n 系统管理
5. **性能监控**：监测首页加载性能和动画流畅度
6. **深色模式**：确保所有视觉效果在深色模式下也表现良好

## 设计原则

本次设计遵循以下原则：

1. **简洁专注** - 聚焦核心产品，避免信息过载
2. **品牌化** - 强化产品线识别，统一视觉语言
3. **可读性** - 保持足够的对比度和留白
4. **响应式** - 多端完美适配
5. **性能** - 流畅不卡顿
