# Warehouse setup and release

The settings page is used to maintain the basic properties and collaboration permissions of the warehouse. Different resource types will display different configuration cards.

## Common settings items

| Setting items | Description |
| --- | --- |
| Visibility | Set public, internal, or private access scope |
| Alias | Update the name displayed to users |
| Members | Manage warehouse members and their permissions |
| Delete warehouse | Delete current resource warehouse |

## Differentiated display by resource type

| Resource type | Typical settings |
| --- | --- |
| Model/Dataset | Metadata Configuration |
| Mirror | Mirror metadata, README |
| Space | Code class metadata, cover, deployment configuration |

## Space deployment

Space is currently the most special resource type in the settings page. It has additional deployment cards and can be used for:

- Configure the parameters required for deployment
- Select linked running resources
- Deliver Space to running environments such as Rune

> ⚠️ Note: The settings page often directly affects resource visibility and team collaboration permissions. It is recommended to be maintained by the warehouse owner or administrator.
