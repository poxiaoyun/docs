# Warehouse details page structure

The resource type and warehouse name will be displayed at the top of the details page, and different tabs will be rendered below based on the resource type and permissions.

## Entry method

It can be accessed by clicking on the card from the model, dataset, image or Space list page.

Typical path example:

- `/moha/models/:organization/:id/content`
- `/moha/datasets/:organization/:id/file`
- `/moha/images/:organization/:id/tags`
- `/moha/spaces/:organization/:id/setting`

## Permission control

The settings page is not available to all users. The current front-end will decide whether to display the "Settings" label based on the following conditions:

- System Administrator
- Warehouse Owner
- Organization Administrator
- Repository administrator or member with read and write permissions

## Page value

- Use a unified operating paradigm to reduce learning costs for different resource types.
- Supports quick switching between resource details, files, discussions and settings.
- Provide fixed work entrances for team collaboration, review and publishing.
