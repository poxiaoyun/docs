## Quick start

---

This article is suitable for users who have just come into contact with Moha. The goal is to become familiar with the model download and model inference of Moha warehouse within 5 minutes. We hope to help you set up your development environment and start model learning, use and development from Moha Warehouse smoothly.

## Environment installation

To use Moha community models in a local development environment, we recommend using Moha SDK to download models. You can install Moha SDK using the following command:

```bash
pip install moha
```

At the same time, we recommend that you install Git and Git LFS, which are necessary tools for model management including uploading.

## Model download

If you are running on a high-bandwidth machine, it is recommended to use the Moha command line tool to download the model. This method supports breakpoint resumption and high-speed model download. For example, you can use the following command to download the Qwen2.5-0.5B-Instruct model to the "model-dir" directory under the current path.

```
moha download --model="Qwen/Qwen2.5-0.5B-Instruct" --local_dir ./model-dir
```

You can also use Moha Python SDK to download the model. This method supports resumable download and high-speed model download:

```
from moha import snapshot_download
model_dir = snapshot_download("Qwen/Qwen2.5-0.5B-Instruct")
```

Since models are stored through Git, you can also download the model locally through git clone after installing Git LFS, for example:

```
git lfs install
git clone https://moha.xiaoshiai.cn/Qwen/Qwen2.5-0.5B-Instruct.git
```

For detailed instructions on model download, please refer to the model download document.

At the same time, if the model is bound to the Moha SDK, it only takes a few lines of code to load the model. At the same time, Moha also supports loading the model through interfaces such as AutoModel. The following is an example of loading a model using AutoModel and pipeline methods:

### Use AutoModel to load the model:

Moha supports native pipeline inference, and is also compatible with AutoModel and Pipeline loading provided by Transformers, Diffusers, etc.

For example, after installing Moha and transformers, you can perform LLM inference through the following code.

```bash
pip install transformers
```

```python
from moha import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen2.5-0.5B-Instruct"

model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype="auto",
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained(model_name)
```

### Use Moha pipeline to load the model:

```python
from moha.pipelines import pipeline
word_segmentation = pipeline('word-segmentation',model='damo/nlp_structbert_word-segmentation_chinese-base')
```

## Model inference

To reason about multiple tasks in different modalities, pipeline is the simplest and fastest method. You can use the out-of-the-box pipeline to perform multiple tasks across different modes. Here is an example of a complete pipeline running:

```python
from moha.pipelines import pipeline
from moha.utils.constant import Tasks

inference_pipeline = pipeline(
    task=Tasks.auto_speech_recognition,
    model='iic/speech_paraformer-large-vad-punc_asr_nat-zh-cn-16k-common-vocab8404-pytorch',
    model_revision="v2.0.4")

rec_result = inference_pipeline('https://isv-data.oss-cn-hangzhou.aliyuncs.com/ics/MaaS/ASR/test_audio/asr_vad_punc_example.wav')
print(rec_result)
```

Moha is compatible with Transformers providing a simple and unified way to load pre-trained instances and tokenizers. This means you can use Moha to load classes like AutoModel and AutoTokenizer. Here is a complete running example of a large language model:

```python
from moha import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen2.5-0.5B-Instruct"

model = AutoModelForCausalLM.from_pretrained(
    model_name,
    torch_dtype="auto",
    device_map="auto"
)
tokenizer = AutoTokenizer.from_pretrained(model_name)

prompt = "Give me a short introduction to large language model."
messages = [
    {"role": "system", "content": "You are Qwen, created by Alibaba Cloud. You are a helpful assistant."},
    {"role": "user", "content": prompt}
]
text = tokenizer.apply_chat_template(
    messages,
    tokenize=False,
    add_generation_prompt=True
)
model_inputs = tokenizer([text], return_tensors="pt").to(model.device)

generated_ids = model.generate(
    **model_inputs,
    max_new_tokens=512
)
generated_ids = [
    output_ids[len(input_ids):] for input_ids, output_ids in zip(model_inputs.input_ids, generated_ids)
]

response = tokenizer.batch_decode(generated_ids, skip_special_tokens=True)[0]
```

For models that are not natively integrated with Moha SDK, you can first download the model from Moha, and then implement model inference through other mainstream libraries. Taking the SDXL-Turbo model as an example, the complete model inference running example is as follows:

```python
from diffusers import AutoPipelineForText2Image
import torch
from moha import snapshot_download

model_dir = snapshot_download("AI-Moha/sdxl-turbo")

pipe = AutoPipelineForText2Image.from_pretrained(model_dir, torch_dtype=torch.float16, variant="fp16")
pipe.to("cuda")

prompt = "A cinematic shot of a baby racoon wearing an intricate italian priest robe."

image = pipe(prompt=prompt, num_inference_steps=1, guidance_scale=0.0).images[0]
image.save("image.png")
```

## Model deployment

You can use Moha Community SwingDeploy to deploy specified models to cloud resources with one click. In addition to supporting the deployment of small models for various tasks, SwingDeploy also supports OpenAI API-compatible calling interfaces for some large language models directly after deployment.

If you want to deploy the model in your own GPU environment, we recommend using vLLM to deploy large language models. First, you need to set environment variables to specify the use of the model on Moha:

```bash
export VLLM_USE_MOHA=True
```

vllm>=0.6 supports vLLM’s built-in tool calling function, deployment command:

```bash
vllm serve Qwen/Qwen2.5-7B-Instruct --enable-auto-tool-choice --tool-call-parser hermes
之后你可以像使用GPT’s tool calling那样来使用它。
```Moha will also support the deployment of more models, so stay tuned.
