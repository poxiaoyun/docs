## CLI tool overview

`moha` is a command line tool provided by Moha SDK, which allows you to complete repository operations without writing code.

### Command overview

| Order | illustrate |
|------|------|
| `moha login` | Log in and save Token |
| `moha logout` | Log out |
| `moha whoami` | View current login status |
| `moha upload` | Upload files folder to repository |
| `moha upload-file` | Upload a single files to repository |
| `moha download` | Download the entire repository |
| `moha download-file` | Download a single files |
| `moha repo-list` | List repository |
| `moha repo-create` | Create repository |
| `moha repo-update` | Update repository information |
| `moha repo-delete` | Delete repository |
| `moha repo-info` | View repository details |
| `moha branch-create` | Create a branch |
| `moha branch-delete` | delete branch |
| `moha branch-list` | list branches |
| `moha space-deploy` | Post Space |
| `moha space-redeploy` | Republish Space |
| `moha space-status` | View Space status |
| `moha ai` | AI intelligent assistant |
| `moha ai-config` | Configure AI Assistant |

### quick help

```bash
# 查看所有命令
moha --help

# 查看特定命令帮助
moha upload --help
moha download --help
```

### General options

All commands support the following authentication options:

| Options | illustrate |
|------|------|
| `--base-url` | Hub API address (or settings `MOHA_ENDPOINT` environment variable) |
| `--token` | Authentication token (or settings `MOHA_TOKEN` environment variable) |
| `--username` | Username (or settings `MOHA_USERNAME` environment variable) |
| `--password` | Password (or settings `MOHA_PASSWORD` environment variable) |

## Authentication command

### `moha login`

Log in and save the token locally.

**usage**:
```bash
moha login [OPTIONS]
```

**Options**:
| Options | illustrate |
|------|------|
| `--username, -u` | Username (enter interactively if not specified) |
| `--password, -p` | Password (enter interactively if not specified) |
| `--base-url` | Hub API address |
| `--token-dir` | Token storage directory (default `~/.moha`) |

**Example**:

```bash
# 交互式登录
moha login

# 指定用户名和密码
moha login --username alice --password secret

# 指定 Hub 地址
moha login \
  --username alice \
  --password secret \
  --base-url https://moha.your-domain
```

### `moha logout`

Log out and delete the local token.

```bash
moha logout
```

### `moha whoami`

View current login status.

```bash
moha whoami
```

## Upload command

### `moha upload` - Upload files folder

Upload the local files folder to repository.

**usage**:
```bash
moha upload <folder> <repo_id> [OPTIONS]
```

**parameter**:
| parameter | illustrate | example |
|------|------|------|
| `<folder>` | Local files folder path | `./my_model` |
| `<repo_id>` | Target repository ID (format: organization/repository name) | `my-org/my-model` |

**Options**:
| Options | illustrate | default value |
|------|------|--------|
| `--repo-type, -t` | repository type (`models`/`datasets`/`spaces`) | `models` |
| `--revision, -r` | branch uploaded to | `main` |
| `--message, -m` | Submit information | none |
| `--ignore, -i` | Ignore matching files (glob pattern, repeatable) | none |
| `--encrypt, -e` | Enable encryption | Disable |
| `--encryption-password` | Encryption password (or settings `MOHA_ENCRYPTION_PASSWORD`) | none |
| `--algorithm, -a` | Encryption algorithm (`AES` / `SM4`) | `AES` |
| `--temp-dir` | Encrypt temporary directory | none |

**Example**:

```bash
# 基础上传
moha upload ./my_model my-org/my-model

# 指定用户名密码
moha upload ./my_model my-org/my-model \
  --username alice \
  --password secret

# 忽略临时文件
moha upload ./my_model my-org/my-model \
  --ignore "*.pyc" \
  --ignore "__pycache__" \
  --ignore "*.tmp"

# 上传数据集
moha upload ./my_dataset my-org/my-dataset \
  --repo-type datasets

# 上传到指定分支
moha upload ./my_model my-org/my-model \
  --revision dev

# 启用 AES 加密上传
moha upload ./my_model my-org/my-model \
  --encrypt \
  --encryption-password secret123

# 使用 SM4 国密算法加密
moha upload ./my_model my-org/my-model \
  --encrypt \
  --encryption-password secret123 \
  --algorithm SM4

# 自定义提交信息
moha upload ./my_model my-org/my-model \
  --message "Upload weights for fine-tuning v2"
```

### `moha upload-file` - Upload a single files

Upload a single files to repository.

**usage**:
```bash
moha upload-file <file> <repo_id> [OPTIONS]
```

**parameter**:
| parameter | illustrate | example |
|------|------|------|
| `<file>` | Local files path | `./config.json` |
| `<repo_id>` | Targetrepository ID | `my-org/my-model` |

**Options**:
| Options | illustrate | default value |
|------|------|--------|
| `--path-in-repo, -p` | The path of files in repository (the default name is files) | filesname |
| `--repo-type, -t` | repository type | `models` |
| `--revision, -r` | branch | `main` |
| `--message, -m` | Submit information | none |
| `--encrypt, -e` | Enable encryption | Disable |
| `--encryption-password` | Encrypted password | none |
| `--algorithm, -a` | encryption algorithm | `AES` |

**Example**:

```bash
# 上传单个文件
moha upload-file ./config.json my-org/my-model

# 指定仓库内路径
moha upload-file ./model.safetensors my-org/my-model \
  --path-in-repo "weights/model.safetensors"

# 上传并加密
moha upload-file ./model.bin my-org/my-model \
  --encrypt \
  --encryption-password secret123

# 自定义提交信息
moha upload-file ./config.json my-org/my-model \
  --message "Update config for v2.0"
```

## Download command

### `moha download` - Download the entire repository

Download all files in repository.

**usage**:
```bash
moha download <repo_id> [OPTIONS]
```

**parameter**:
| parameter | illustrate | example |
|------|------|------|
| `<repo_id>` | repository ID | `ai-lab/qwen2-7b` |

**Options**:
| Options | illustrate | default value |
|------|------|--------|
| `--local-dir, -o` | Downloaded to local directory | current directory |
| `--repo-type, -t` | repository type (`models`/`datasets`/`spaces`) | `models` |
| `--revision, -r` | branch or tags | none |
| `--include` | Download only matching files (glob pattern, repeatable) | Unlimited |
| `--ignore, -i` | Ignore matching files (glob pattern, repeatable) | none |
| `--quiet, -q` | Disable progress bar | show progress |

**Example**:

```bash
# 基础下载
moha download ai-lab/qwen2-7b

# 下载到指定目录
moha download ai-lab/qwen2-7b --local-dir ./my_models

# 只下载权重文件
moha download ai-lab/qwen2-7b \
  --include "*.safetensors"

# 忽略大文件
moha download ai-lab/qwen2-7b \
  --ignore "*.bin"

# 下载数据集
moha download ai-lab/my-dataset \
  --repo-type datasets \
  --local-dir ./datasets

# 下载特定版本
moha download ai-lab/qwen2-7b \
  --revision v1.0 \
  --local-dir ./qwen2-v1

# 静默下载
moha download ai-lab/qwen2-7b --quiet

# 从私有仓库下载
moha download my-org/private-model \
  --username alice \
  --password secret
```

### `moha download-file` - Download a single files

Only download specific files in repository.

**usage**:
```bash
moha download-file <repo_id> <filename> [OPTIONS]
```

**parameter**:
| parameter | illustrate | example |
|------|------|------|
| `<repo_id>` | repository ID | `ai-lab/qwen2-7b` |
| `<filename>` | files path within repository | `config.json` |

**Options**:
| Options | illustrate |
|------|------|
| `--local-dir, -o` | Downloaded to local directory |
| `--repo-type, -t` | repository type |
| `--revision, -r` | branch or tags |
| `--quiet, -q` | Disable progress bar |

**Example**:

```bash
# 下载单个文件
moha download-file ai-lab/qwen2-7b config.json

# 下载到指定目录
moha download-file ai-lab/qwen2-7b config.json \
  --local-dir ./configs

# 下载特定版本的文件
moha download-file ai-lab/qwen2-7b config.json \
  --revision v1.0
```

## repository management commands

### `moha repo-list` - List repository

**usage**:
```bash
moha repo-list [OPTIONS]
```

**Options**:
| Options | illustrate | default value |
|------|------|--------|
| `--repo-type, -t` | repository type (`models`/`datasets`/`spaces`) | `models` |
| `--scope, -s` | Query range (`public`/`create`/`favorite`/`organization`) | `public` |
| `--organization, -o` | Organization name (required when scope=organization) | none |
| `--mine, -m` | Show only the repository I created (only valid when scope=organization) | no |

**Example**:

```bash
# 列出公开模型
moha repo-list

# 列出公开数据集
moha repo-list --repo-type datasets

# 列出我创建的模型
moha repo-list --scope create

# 列出我收藏的模型
moha repo-list --scope favorite

# 列出组织内的模型
moha repo-list --scope organization --organization my-org

# 列出组织内我创建的模型
moha repo-list --scope organization --organization my-org --mine
```

### `moha repo-create` - Create repository

**usage**:
```bash
moha repo-create <repo_id> [OPTIONS]
```

**Options**:
| Options | illustrate | default value |
|------|------|--------|
| `--repo-type, -t` | repository type | `models` |
| `--description, -d` | repositoryDescription | none |
| `--visibility, -v` | Visibility (`public`/`internal`/`private`) | `internal` |
| `--license` | License (repeatable) | none |
| `--tasks` | Task type (repeatable) | none |
| `--languages` | Language (repeatable) | none |
| `--tags` | tags (repeatable) | none |
| `--frameworks` | Frame (repeatable) | none |
| `--base-model` | Base model (repeatable, format `组织/模型名`) | none |
| `--relationship` | Relationship with the basic model (`adapter`/`finetune`/`quantized`/`merge`/`repackage`) | none |

**Example**:

```bash
# 创建模型仓库
moha repo-create my-org/my-model

# 创建数据集仓库
moha repo-create my-org/my-dataset --repo-type datasets

# 创建含描述的公开仓库
moha repo-create my-org/my-model \
  --description "Fine-tuned Qwen2 model" \
  --visibility public

# 创建包含元数据的仓库
moha repo-create my-org/my-model \
  --license apache-2.0 \
  --tasks text-generation \
  --languages zh \
  --frameworks pytorch \
  --base-model ai-lab/qwen2-7b \
  --relationship finetune
```

### `moha repo-update` - Update repository

Update repository description, visibility, metadata and other information.

```bash
moha repo-update my-org/my-model \
  --description "Updated description" \
  --visibility public
```

### `moha repo-delete` - Remove repository

```bash
# 删除仓库（需确认）
moha repo-delete my-org/old-model

# 跳过确认
moha repo-delete my-org/old-model --yes
```

### `moha repo-info` - View repository details

```bash
moha repo-info my-org/my-model
moha repo-info my-org/my-dataset --repo-type datasets
```

## AI intelligent assistant

### `moha ai` - Start AI Assistant

Manage repository using natural language.

```bash
# 交互模式
moha ai

# 非交互模式（单条指令）
moha ai --prompt "列出所有模型"

# 指定模型
moha ai --model gpt-4
```

**Options**:
| Options | illustrate |
|------|------|
| `--prompt, -p` | Execute a single instruction directly |
| `--api-base` | AI API address (or settings `MOHA_AI_API_BASE`) |
| `--api-key` | AI API key (or settings `MOHA_AI_API_KEY`) |
| `--model, -m` | Model name (or settings `MOHA_AI_MODEL`) |
| `--no-stream` | Disable streaming output |

### `moha ai-config` - Configure AI Assistant

```bash
# 配置 API
moha ai-config --api-base https://api.example.com --api-key sk_xxx --model gpt-4

# 查看当前配置
moha ai-config --show
```

## troubleshooting

### Issue 1: Authentication failed

```bash
# 重新登录
moha login --username alice --password secret

# 或明确指定 Token
moha upload ./my_model my-org/my-model \
  --token sk_xxxxxxxxxxxx
```

### Problem 2: Large files upload failed

```bash
# SDK 自动处理大文件分块上传
# 如果中途中断，重新运行命令会自动续传
moha upload ./my_model my-org/my-model
```

### Question 3: files filtering does not take effect

```bash
# 注意引号防止 shell 扩展
# upload 使用 --ignore 过滤
moha upload ./my_model my-org/my-model \
  --ignore "*.pyc" \
  --ignore "__pycache__"

# download 使用 --include 选择文件，--ignore 排除文件
moha download ai-lab/qwen2-7b \
  --include "*.safetensors" \
  --ignore "*.bin"
```

## best practices

### 1. Safety regulations

```bash
# ✗ 不要在命令行暴露密码
moha upload ./model org/repo --password secret

# ✓ 使用环境变量或提前 login
export MOHA_TOKEN=sk_xxxxxxxxxxxx
moha upload ./model org/repo
```

### 2. files selection

```bash
# 忽略不必要文件
moha upload ./my_model org/repo \
  --ignore "*.pyc" \
  --ignore "__pycache__" \
  --ignore ".git"
```

### 3. Version management

```bash
# 上传时指定清晰的提交信息
moha upload ./my_model org/repo \
  --message "Release v1.0: Added LoRA support"
```
