## Encryption overview

Moha SDK supports password-based files encrypted upload. Core capabilities include:

- Encrypt files content when uploading
- Supports `AES` and `SM4` algorithms
- Automatically identify and decrypt the encrypted files header when downloading (correct password required)
- Support for providing default password via `MOHA_ENCRYPTION_PASSWORD` environment variable

### Algorithm description

| algorithm | illustrate | Applicable scenarios |
|------|------|----------|
| `AES` | Internationally applicable, good performance | Recommended by default |
| `SM4` | National secret algorithm | Scenarios requiring national confidentiality compliance |

## CLI encrypted upload

### Upload the files folder and encrypt it

```bash
moha upload \
  -t models \
  -e \
  --encryption-password "your-password" \
  -a AES \
  -m "Encrypted upload" \
  "my-org/my-model" \
  ./model
```

### Upload the leaflet files and encrypt it

```bash
moha upload-file \
  -t models \
  -e \
  --encryption-password "your-password" \
  -a SM4 \
  -m "Upload encrypted weight" \
  "my-org/my-model" \
  ./model.safetensors \
  model.safetensors
```

### Provide password using environment variables

```bash
export MOHA_ENCRYPTION_PASSWORD="your-password"

moha upload \
  -t datasets \
  -e \
  -a AES \
  "my-org/my-dataset" \
  ./dataset
```

> `-e/--encrypt` only starts the encryption process; the password comes from `--encryption-password` or `MOHA_ENCRYPTION_PASSWORD`.

## Python API encrypted upload

### Single files encrypted upload

```python
from xiaoshiai_hub import upload_file

upload_file(
    path_file="./model.safetensors",
    path_in_repo="weights/model.safetensors",
    repo_id="my-org/my-model",
    repo_type="models",
    encryption_password="your-password",
    algorithm="AES",
    commit_message="Upload encrypted model weight"
)

print("✓ 文件已加密上传")
```

### files folder encryption upload

```python
from xiaoshiai_hub import upload_folder

upload_folder(
    folder_path="./dataset",
    repo_id="my-org/my-dataset",
    repo_type="datasets",
    encryption_password="your-password",
    algorithm="SM4",
    ignore_patterns=["*.tmp", "__pycache__", "*.pyc"],
    commit_message="Upload encrypted dataset"
)

print("✓ 文件夹已加密上传")
```

### Key parameters

| parameter | effect |
|------|------|
| `encryption_password` | Encrypted password |
| `algorithm` | Encryption algorithm, supports `"AES"` / `"SM4"` |
| `ignore_patterns` | Ignore files mode when uploading files folder |

## Download and decrypt

When encrypted files is downloaded, the SDK will automatically identify and perform decryption based on the files header. You need to provide the correct password in the runtime environment.

### CLI download

```bash
export MOHA_ENCRYPTION_PASSWORD="your-password"

moha download \
  -t models \
  "my-org/my-model" \
  -o ./downloads
```

### Python download

```python
import os
from xiaoshiai_hub import snapshot_download, moha_hub_download

os.environ["MOHA_ENCRYPTION_PASSWORD"] = "your-password"

# 下载整个仓库
repo_path = snapshot_download(
    repo_id="my-org/my-model",
    repo_type="models",
    local_dir="./downloads/model"
)
print(f"✓ 仓库已下载: {repo_path}")

# 下载单个文件
file_path = moha_hub_download(
    repo_id="my-org/my-model",
    filename="weights/model.safetensors",
    repo_type="models",
    local_dir="./downloads/single"
)
print(f"✓ 文件已下载: {file_path}")
```

## DataKey and KMS

`HubClient` provides `generate_data_key()` for requesting the data key from the server:

```python
from xiaoshiai_hub import HubClient

client = HubClient(token="sk_xxxxx")

data_key = client.generate_data_key(
    algorithm="AES",
    password="your-password"
)

print(type(data_key).__name__)  # DataKey
print(len(data_key.plaintext_key))
print(data_key.encrypted_key[:24] + "...")
```

`DataKey` is a simple data structure with the following fields:

```python
@dataclass
class DataKey:
    plaintext_key: bytes
    encrypted_key: str
```

> In the current SDK, `DataKey` does not have `generate()` / `save()` / `from_file()` / `revoke()` methods.

## Practical workflow

### Scenario 1: Uploading sensitive data sets

```python
from xiaoshiai_hub import upload_folder

upload_folder(
    folder_path="./sensitive_dataset",
    repo_id="my-org/sensitive-data",
    repo_type="datasets",
    encryption_password="team-secret",
    algorithm="AES",
    ignore_patterns=["*.tmp", "*.log", "__pycache__"],
    commit_message="Encrypted dataset upload"
)

print("✓ 敏感数据集已加密上传")
```

### Scenario 2: Teams share encryption models

```python
import os
from xiaoshiai_hub import upload_file, snapshot_download

# 管理员上传（加密）
upload_file(
    path_file="./model.safetensors",
    path_in_repo="model.safetensors",
    repo_id="my-team/shared-model",
    encryption_password="team-password",
    algorithm="AES",
    commit_message="Upload encrypted team model"
)

# 团队成员下载（解密）
os.environ["MOHA_ENCRYPTION_PASSWORD"] = "team-password"
model_path = snapshot_download(
    repo_id="my-team/shared-model",
    repo_type="models"
)
print(f"✓ 团队模型已下载: {model_path}")
```

### Scenario 3: National secret algorithm upload

```python
from xiaoshiai_hub import upload_folder

upload_folder(
    folder_path="./compliance_data",
    repo_id="my-org/compliance-dataset",
    repo_type="datasets",
    encryption_password="compliance-secret",
    algorithm="SM4",
    commit_message="Upload encrypted by SM4"
)
```

## Security advice

### Password management

- Do not hardcode the encryption password into the repository code.
- Prefer using environment variables or CI Secret injection.
- It is recommended to use different passwords for different projects to reduce lateral risks.

### CI/CD examples (GitHub Actions)

```yaml
name: Upload Encrypted Artifacts

on:
  push:
    branches: [main]

jobs:
  upload:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.10'

      - name: Install SDK
        run: pip install xiaoshiai-hub

      - name: Upload encrypted folder
        env:
          MOHA_TOKEN: ${{ secrets.MOHA_TOKEN }}
          MOHA_ENCRYPTION_PASSWORD: ${{ secrets.MOHA_ENCRYPTION_PASSWORD }}
        run: |
          moha upload \
            -t models \
            -e \
            -a AES \
            -m "CI encrypted upload" \
            my-org/my-model \
            ./model
```

## Troubleshooting

### 1. Error when uploading: Missing encryption password

Symptom: The encryption parameter `-e` or settings is enabled, but no password is provided.

solve:

```bash
export MOHA_ENCRYPTION_PASSWORD="your-password"
# 或在命令中显式传 --encryption-password
```

### 2. Decryption failed after downloading

Symptom: Download is successful but content cannot be decrypted.

solve:

- Check if `MOHA_ENCRYPTION_PASSWORD` is correct.
- Make sure you use the same password for upload and download.
- Verify that the algorithm and files have not been tampered with.

### 3. Incorrect algorithm parameters

Symptom: An unsupported algorithm string was passed in.

Resolution: Only use `AES` or `SM4`.
