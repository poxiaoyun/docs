## Overview
Through the standardized version release process, the team can record the source, data set and evaluation indicators of each training product, while providing reproducible model assets to the community.

## Preconditions
- Model training products have been uploaded to object storage or modelrepository.
- Has `ModelPublisher` permission.

## Operation steps
1. **Create version**: Click "Publish" in the "Model Version" list, fill in the version number, change description and visible range.
2. **Associated Data Set**: Select the data set version used for this training to implement bloodline tracking.
3. **Upload product**: Provide model weights, configuration files and inference scripts.
4. **Submit for review**: If the review process is started, the version will enter the `Reviewing` status.

## Example
```bash
rune model publish \
  --model llama2-7b \
  --version 1.1.0 \
  --artifact s3://models/llama2-7b-1.1.0
```

## FAQ
- **Q: How to name the version number? **
**A:** It is recommended to use `主版本.次版本.修订` semantic naming to facilitate automatic comparison.
- **Q: Can I settings paid visibility? **
**A:** The enterprise version supports additional subscription strategies for the model version, please contact business activation.
