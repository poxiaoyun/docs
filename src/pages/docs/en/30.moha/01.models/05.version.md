## Model version management

Manage multiple versions of the model, supporting release, rollback and version comparison.

## Version mechanism

Moha Warehouse uses Git branches and tags to implement model version management:

- **Branch**: used for versions under development, such as `main`, `dev`, `feature-xxx`
- **tag**: used for released stable versions, such as `v1.0`, `v2.0-beta`

## Operation Guide

### View version

1. Enter the model details page
2. Switch to the **Version** tab
3. View all branches and tags

### Create new version

1. Complete model training or update locally
2. Push new files to the warehouse
3. Create a label to mark the version number

```bash
git tag v1.0 -m "Release v1.0"
git push origin v1.0
```

### Switch version

Select the target branch or label in the version drop-down box at the top of the model details page to view the files and information of that version.

:::tip
It is recommended to use semantic version numbers (such as v1.0.0). The major version number indicates incompatible changes, minor indicates new features, and patch indicates repairs.
:::
