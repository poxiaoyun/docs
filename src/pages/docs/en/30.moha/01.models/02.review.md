## Overview
The review process ensures that models published to Moharepository meet security and compliance requirements. This chapter describes how reviewers assign tasks, check indicators, and make decisions.

## Preconditions
- Model version is in `Reviewing` status.
- Reviewers have the `ModelReviewer` role.

## Operation steps
1. **Receive tasks**: View the list to be reviewed in the "Audit Center" and receive the specified model version.
2. **Check content**: Check model introduction, data source, License and security statement.
3. **Automatic detection**: Trigger automated scripts to scan sensitive information and dependency vulnerabilities.
4. **Decision**: Select pass/reject, fill in the comments and @ the publisher.

## Example
```bash
rune model review \
  --model llama2-7b \
  --version 1.1.0 \
  --result approve \
  --comment "指标达标，可对外发布"
```

## FAQ
- **Q: How to assign reviewers? **
**A:** Click "Assign" in the review details and select the new person in charge.
- **Q: Automatic detection failed? **
**A:** Check the log to confirm that the detection script image is accessible; try again or contact the platform engineer if necessary.
