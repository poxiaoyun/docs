## Overview

Publish, update and rollback model versions.

## Version number specification

It is recommended to use semantic versioning (Semantic Versioning):

- **Major version number**: Incompatible API modification
- **Minor version number**: Backwards compatible functional additions
- **Revision number**: Backward compatibility bug fixes

Example: `v1.2.3`

## Release new version

1. Enter the model details page
2. Click "Publish new version"
3. Upload new modelfiles
4. Fill out the release notes and change log

## Version comparison

See the differences between versions:

- files size changes
- Comparison of performance indicators
- Changes in the number of parameters

## rollback version

If something goes wrong with the new version, you can quickly roll back to the stable version:

1. Enter "Version History"
2. Select the version to roll back
3. Click "Set as default version"
