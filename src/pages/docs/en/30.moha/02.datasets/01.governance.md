## Overview
A high-quality data governance process ensures that data sources are legal, safe, and controllable. This section provides end-to-end guidance from onboarding and desensitization to publishing.

## Preconditions
- Data has undergone copyright and compliance assessment.
- Has `DatasetMaintainer` permission.

## Operation steps
1. **Onboarding check**: Upload the dataset's basic information, including source, license, and size.
2. **Desensitization**: Use the official script to mask sensitive fields and record the processing result.
3. **Version freeze**: Create a version tag and lock the hash to prevent later modification.
4. **Permission control**: Set the dataset to public or private visibility, or restrict it to specific organizations.

## Example

```bash
rune dataset publish \
  --name fintech-train \
  --version 2.0.0 \
  --manifest manifest.yaml
```

## FAQ
- **Q: How do I synchronize large volumes of data?**
  **A:** It is recommended to use direct transfer from object storage or an offline import tool to avoid browser upload timeouts.
- **Q: Can files still be modified after the version is published?**
  **A:** The original files cannot be modified. You can create a new version based on that release and record the changes.
