## Dataset governance guidelines

Guidelines and best practices for dataset management.

## Naming convention

| Item | Rule | Example |
|------|------|------|
| Dataset name | Lowercase letters, numbers, and hyphens | `chinese-qa-v2` |
| File naming | Clear naming that includes a version or date | `train-2025-01.parquet` |
| Directory structure | Split directories by data type or usage | `train/`, `test/`, `validation/` |

## Data quality requirements

- **Completeness**: Ensure that all fields have values, mark null values and missing reasons
- **Consistency**: Use a unified field format, such as date format and encoding.
- **Accuracy**: Ensure annotations are correct and the data has no obvious errors.
- **Timeliness**: Record the data collection time and last update time.

## Privacy and Security

- Desensitize personal information such as names, phone numbers, and ID numbers.
- Indicate data sources and usage restrictions in the dataset card
- Set sensitive datasets to private visibility.

## License selection

Commonly used open data licenses:

| License | Description |
|--------|------|
| CC BY 4.0 | Free use is allowed as long as the source is credited. |
| CC BY-SA 4.0 | Use is allowed, and derivative works must keep the same license. |
| CC BY-NC 4.0 | Non-commercial use is allowed. |
| Apache 2.0 | A permissive license suitable for code-oriented datasets. |
