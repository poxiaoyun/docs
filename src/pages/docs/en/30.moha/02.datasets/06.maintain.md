## Dataset maintenance

Manage and maintain daily data sets to ensure data quality.

## Version management

Use Git branches and tags to manage dataset versions:

- **Branch**: `main` is the official version, create other branches for development
- **Tags**: Tag stable versions (such as `v1.0`, `v2.0`)

## Data update

###Append data

```bash
cd my-dataset
cp /path/to/new-data/*.parquet .
git add .
git commit -m "追加 2025-01 训练数据"
git push origin main
```

### Rollback version

If there are problems with the new data, you can roll back to the previous version:

```bash
git revert HEAD
git push origin main
```

## Cleaning and archiving

- **Clean useless files**: Delete expired or useless data files
- **Archive historical version**: Archive old version data to an independent branch

:::warning
Git history for large data sets can take up a lot of storage space. It is recommended to regularly clean up historical large files that are no longer needed.
:::
