## Overview

Cluster onboarding is usually handled in stages so that tenants only see stable capacity. The workflow starts with connectivity and inventory checks, then moves through resource pool and flavor setup, and ends with publication to downstream users.

## Recommended Sequence

- Create the cluster entry and validate the kubeconfig connection.
- Review nodes, runtime dependencies, storage services, and monitoring components.
- Define resource pools and flavors so capacity can be exposed in a controlled way.
- Publish the cluster only after tenant quotas and supporting system services are ready.
