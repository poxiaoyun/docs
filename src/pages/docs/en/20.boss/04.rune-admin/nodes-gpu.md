## Overview

The node and GPU dashboards are the main troubleshooting surfaces when you suspect capacity pressure, machine instability, or imbalance across accelerator devices.

## Suggested Reading Order

- Use the cluster overview first to confirm whether the signal is broad or localized.
- Open the node dashboard when you suspect host-level issues such as pressure, readiness loss, or uneven memory use.
- Open the GPU dashboard when you suspect accelerator saturation, low utilization, or card-level imbalance.
- Move to resource or log pages when you need to connect machine signals to a specific workload or object.
