## Overview

Cluster Overview is the top-level page after entering a specific Rune cluster from Boss. It presents baseline monitoring panels and acts as the first checkpoint before operators drill down into nodes, storage, logs, or Kubernetes objects.

## Typical Uses

- Confirm whether an issue is cluster-wide or isolated to a smaller surface area.
- Check general CPU, memory, node, and core service health before making changes.
- Use overview findings to decide whether to inspect nodes, GPU panels, resource objects, or logs next.
