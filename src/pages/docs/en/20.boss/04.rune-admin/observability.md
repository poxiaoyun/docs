## Overview

The observability area gathers the pages that operators use to inspect cluster behavior from different angles: metrics, events, real-time logs, and scheduler configuration.

## What This Area Covers

- Use logs for query-based troubleshooting, label filtering, and live streams.
- Use monitoring and event views when you need historical context around failures or load changes.
- Use the scheduler page carefully when policy order or plugin configuration needs adjustment.
- Validate scheduling changes in lower-risk environments before applying them to production clusters.
