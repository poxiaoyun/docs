## Overview

A flavor is the platform-facing resource package that maps to a cluster configuration. Users select flavors during deployment instead of requesting raw CPU, memory, and accelerator values one by one.

## How to Use This Page

- Check which clusters expose the flavor you need.
- Compare CPU, memory, GPU, and accelerator characteristics before creating the instance.
- Confirm that the current workspace quota includes the target flavor.
- Use standard team flavors whenever possible to keep cost and troubleshooting predictable.
