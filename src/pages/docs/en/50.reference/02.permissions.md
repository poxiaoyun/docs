## Overview

The permission model explains how roles, scopes, and product context combine to grant or deny access across the platform. It is useful for administrators, integrators, and feature teams alike.

## Useful Questions This Page Answers

- Which actions are platform-wide and which are tenant or workspace scoped?
- How do administrator, developer, and member roles differ across products?
- Why can the same user see different resources after switching tenant or workspace context?
