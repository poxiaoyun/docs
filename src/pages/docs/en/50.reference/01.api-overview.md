## Overview

This page is the technical starting point for readers who need to understand how the platform’s backend services are divided and how requests move across identity, gateway, workload, and repository boundaries.

## What to Expect

- A domain-level view of the main backend services rather than endpoint-by-endpoint detail.
- How authentication, tenant scope, and product context shape request behavior.
- The relationship between user-facing products such as Rune, Moha, ChatApp, and Boss and the services behind them.
