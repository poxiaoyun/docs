## Scope of application

When using resource monitoring, you usually need to deploy NPU Exporter and connect the indicators to Prometheus or Telegraf.

## Use constraints (key points for implementation)

- **Driver upgrade sequence**: Exporter will periodically call the driver interface; before upgrading the driver, it is recommended to stop the business first, and then stop Exporter.
- **K8s version**: K8s 1.24+ scenarios may require additional preparation of `cri-dockerd` (depending on your cluster runtime and deployment method).
- **Dynamic library permissions**: DCMI dynamic libraries and their parent directories need to meet stricter owner and write permission constraints (it is recommended to be managed by root to avoid group/other write permissions).
- **Container deployment risk**: Containerized operation often requires privileges and socket mounting. Binary deployment may be considered when security requirements are high.

## Deployment method 1: Containerization (DaemonSet, recommended)

### 1) Confirm that the image exists

Check the image on the compute node (example):

```bash
docker images | grep npu-exporter
```

or containerd:

```bash
ctr -n k8s.io c ls | grep npu-exporter
```

### 2) Select and modify YAML

Copy the YAML in the decompressed directory of the software package to the K8s management node.

- The difference between Docker / containerd scenarios is mainly reflected in the startup parameter `-containerMode` and the mounting path of the runtime socket.
- If you need to adjust the port/update cycle/log, etc., you can modify the startup parameters (such as `-port`, `-updateTime`, `-logFile`, etc.).

### 3) Startup and inspection

```bash
kubectl apply -f npu-exporter-v{version}.yaml
kubectl get pod -n npu-exporter
```

If `CrashLoopBackOff` appears, first check whether the log directory permissions, DCMI dynamic library permissions and socket mounting path are correct.

## Deployment method two: binary (when security requirements are high)

The idea is to put `npu-exporter` in a fixed directory and delay startup through systemd service/timer (to avoid the device not being ready).

- It is recommended to configure the correct owner/permissions for the log directory
- To obtain container-related information, you may need to grant capabilities to the binary and restart the service after the change.
