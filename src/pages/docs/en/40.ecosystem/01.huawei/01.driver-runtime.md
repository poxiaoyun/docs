## Overview

This page focuses on the Ascend NPU **driver and firmware** installation process and verification method in the **openEuler scenario**. It is suitable for physical machine/container deployment (installation on the physical machine) and virtual machine scenarios (physical machine installation of driver + firmware, virtual machine installation of driver only).

This article is organized "delivery-oriented" based on the installation guide of the Ascend community CANN Community Edition, so that it can be quickly referenced in the ecological documents.

## Preconditions

- It has been confirmed that the node OS is openEuler and the kernel version is compatible with the target driver version.
- The driver package and firmware package (`.run` installation package) have been prepared and can be accessed on the target machine
- Confirmed installation/upgrade scenarios: first time installation or overlay installation (determine installation order)
- Have root privileges (installation usually requires root)

:::tip If you still need to install CANN (ToolKit/Kernel/Runtime), please continue according to the official "Installing CANN" chapter after the driver/firmware is completed and verified.:::

## Operation steps

### 1) Confirm whether the driver has been installed

You can first execute the following command to check whether the current environment already has driver information:

```bash
npu-smi info
```

If NPU/driver related information can be returned, it means it has been installed; otherwise, it is considered not installed or loaded successfully.

### 2) Determine the installation order (first installation vs overlay installation)

- **First time installation** (the device is not installed at the factory / has been uninstalled and then reinstalled): Install in the order of **Driver → Firmware**
- **Overlay installation** (installed and not uninstalled, reinstall/upgrade): Install in the order of **Firmware → Driver**

### 3) Prepare the running user (default HwHiAiUser)

The default running user/user group during driver/firmware installation is `HwHiAiUser`. If the user/group does not exist in the environment, it is recommended to create it in advance:

```bash
groupadd HwHiAiUser
useradd -g HwHiAiUser -d /home/HwHiAiUser -m HwHiAiUser -s /bin/bash
```

If you need to use another running user, you need to specify it explicitly during installation (see the "Installing the Driver" step below).

:::warning If `--install-for-all` is not used when installing the driver/firmware, the user running the subsequent CANN software package usually needs to be in the same user group as the user running the driver/firmware, otherwise permissions/access issues may occur.:::

### 4) Install dependencies (openEuler)

Refresh yum cache:

```bash
yum makecache
```

Install compilation/kernel related dependencies (the example is based on the current kernel):

```bash
yum install -y make dkms gcc kernel-headers-$(uname -r) kernel-devel-$(uname -r)
```

If dependencies are missing or dkms-related errors occur, you usually need to check the software source availability or fill in the dependencies according to the official "Installation Dependencies/Driver Error Handling" section.

### 5) Install drivers and firmware

Upload the driver package and firmware package to any directory on the target machine (example: `/home`), enter the directory and grant execution permissions:

```bash
chmod +x Ascend-hdk-<chip_type>-npu-driver_<version>_linux-<arch>.run
chmod +x Ascend-hdk-<chip_type>-npu-firmware_<version>.run
```

The installation path is usually `/usr/local/Ascend` (subject to the actual installation echo).

#### Install driver

Example (full installation + effective for all users):

```bash
./Ascend-hdk-<chip_type>-npu-driver_<version>_linux-<arch>.run --full --install-for-all
```

If you do not use the default running user/user group, you can specify it when installing the driver:

```bash
./Ascend-hdk-<chip_type>-npu-driver_<version>_linux-<arch>.run \
  --full \
  --install-username=<username> \
  --install-usergroup=<usergroup>
```

#### Install firmware

```bash
./Ascend-hdk-<chip_type>-npu-firmware_<version>.run --full
```

After the installation is complete, follow the installation echo prompts to decide whether you need to restart:

```bash
reboot
```

## Verification method

After restarting (or following the prompts to complete loading), execute:

```bash
npu-smi info
```

If driver-related information can be returned, it usually means that the driver was loaded successfully.

If you are in a container deployment scenario, it is recommended to add "minimum verification within the container": start a container image with NPU visibility, and execute the minimum operator/inference sample in the container (subject to your platform's built-in image and verification script).

## FAQ

### npu-smi has no output or cannot see the device

- Confirm whether to restart according to the installation echo requirements
- Confirm whether the driver installation is successful (key prompts indicating successful installation should appear in the installation echo)
- Check whether the dependencies are complete (dkms, kernel headers/devel, etc.)

### During the installation process, it is prompted that Linux tools / dkms / dependencies are missing.

- First check if the yum source is available: `yum makecache`
- Then complete the dependencies: `make` / `dkms` / `gcc` / `kernel-headers` / `kernel-devel`

### Inconsistent coverage installation order causes exceptions

- Overlay installation that has been installed but not uninstalled, generally in the order of **firmware → driver**
- For first-time installation, generally follow the sequence **Driver → Firmware**
