## Monitoring indicators

DCU-Exporter exposes DCU monitoring indicators through HTTP interface: default port `16080`, default path `/metrics`.

Access example: `http://<host>:16080/metrics`

## Installation and deployment

K8s clusters are generally deployed in the form of DaemonSet:

```bash
kubectl create -f dcu-exporter-v2.yaml
```

### Port and indicator collection configuration

- If you need to customize the port, you can modify `containerPort`, `prometheus.io/port` and `service port` in `dcu-exporter-v2.yaml`.
- All indicators are collected by default. You can use the environment variable `ENABLE_METRICS` to specify the indicators that need to be collected (separated by English commas).
- The indicator collection interval can be set with `PULSE` (seconds).
- Log output level/level/switch is configured through `LOG*` environment variables.

## Deployment verification

View Pods:

```bash
kubectl get pod -n kube-system -owide | grep dcu-exporter
```

View metrics via curl:

```bash
curl localhost:16080/metrics
```

If the cluster has deployed Prometheus, you can query relevant indicators in Prometheus to verify the crawling situation.

## Grafana Panel

Grafana Dashboard is available from the developer community for visual presentation.

## Physical machine environment installation and deployment

In addition to containerized deployment, users can install and deploy DCU-Exporter on physical machines through one-click installation (the DCU driver must be installed).

- Download `dcu-exporter.run` from the community
- Perform the installation:

```bash
bash dcu-exporter.run
```

Installation directory: `/opt/dcu-exporter`

Startup example:

```bash
/opt/dcu-exporter/dcu-exporter -h
```
