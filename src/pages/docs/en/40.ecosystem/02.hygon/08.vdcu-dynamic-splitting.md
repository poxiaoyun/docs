## Overview

The Kubernetes cluster uses the device plug-in DCU-Device-Plugin and vDCU-Scheduler (or HAMi scheduler) to implement dynamic segmentation and automatic recycling of vDCU on the node.

The k8s task makes vDCU dynamic splitting requests through the resource fields `hygon.com/dcunum`, `hygon.com/dcucores`, `hygon.com/dcumem`, eliminating the need to manually divide vDCU instances before using vDCU.

## Installation and deployment

It is recommended to use the DCU-Device-Plugin in HAMi mode to manage vDCU after labeling the nodes accordingly.

```bash
# 固定标签：部署设备插件时需要查询此标签
kubectl label nodes <node-name> dcu=on

kubectl apply -f vdcu-admission-webhook.yaml
kubectl apply -f vdcu-scheduler.yaml
kubectl apply -f k8s-dcu-plugin-hami.yaml
```

## Deployment verification

View related Pods:

```bash
kubectl get pod -n kube-system -owide | grep vdcu
```

View node resource registration:

```bash
kubectl describe node <node-name>
```

Example: A Capacity field showing `hygon.com/dcunum: 4` indicates that the node has 4 available vDCU resources.

## Application examples

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: vdcu-pytorch-demo
spec:
  containers:
    - name: vdcu-pytorch-demo
      image: image.sourcefind.cn:5000/dcu/admin/base/pytorch:2.1.0-ubuntu22.04-dtk24.04.2-py3.10
      command: ["/bin/bash", "-c", "--"]
      args: ["sleep infinity & wait"]
      resources:
        limits:
          hygon.com/dcunum: 1      # 申请 1 个 vDCU
          hygon.com/dcucores: 60   # 指定 vDCU 算力为 60%
          hygon.com/dcumem: 3000   # 指定 vDCU 显存为 3000MB
```

Usage restrictions: Each container can apply for up to 1 vDCU. When applying for more than 1 vDCU, the k8s resource pool will reduce the number of applications and only 1 vDCU will take effect.

## Note on uninstallation/redeployment

If you uninstall or redeploy the vDCU dynamic segmentation component, in addition to uninstalling the above three components, you also need to manually clean up the legacy resources:

```bash
kubectl delete secrets -n kube-system admission-webhook-dcu-certs
kubectl delete mutatingwebhookconfigurations mutating-webhook-dcu-cfg
```
