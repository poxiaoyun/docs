## 2. Driver installation process

### 2.1 Install dependency packages

```bash
centos: yum install -y rpm-build gcc-c++ cmake automake elfutils-libelf-devel libdrm libdrm-devel pciutils
ubuntu: apt install -y gcc g++ cmake automake libelf-dev libdrm_amdgpu1 or libdrm-amdgpu1 libtinfo5 pciutils libdrm-dev
centos: yum install -y kernel-devel-`uname -r` kernel-modules-extra
ubuntu: apt install -y linux-headers-`uname -r` linux-modules-extra
```

### 2.2 Install the driver

```
Installation steps
1) Download the driver from: https://cancon.hpccube.com:65024/6/main/latest%E9%A9%B1%E5%8A%A8
2) Grant execute permission: chmod +x rock*.run
3) Install the driver:
./rock*.run
4) Restart the driver service:
systemctl restart hymgr

Notes:
1) You can inspect additional installation parameters with `./rock*.run -h`.
2) You do not need to uninstall the previous driver before installing a new one. The installer removes the old driver during the installation process.
3) During installation you may see `Do you want to remove legacy hymgr config file? [Y/n]:`. Choosing `yes` regenerates `/etc/hymgr.cfg`; choosing `no` keeps the existing file.
4) In K100 or K100-AI environments you may be prompted with `Update vbios? [y/N]:`. During the first installation, it is recommended to update the VBIOS and then reboot the physical machine. For later driver upgrades, you can choose `N`. In cloud scenarios, where physical node restarts are strictly controlled for stability reasons, you may skip the VBIOS update.
```

## Remarks

1. New driver versions require `cmake` later than 3.2 and `gcc` later than 7.3.
2. The kernel version must match the `kernel-devel` version.
3. Drivers before 6.2.* expose the module name `hydcu`, while drivers after 6.3.* expose `hycu`.
4. With 6.3.* drivers, when calling Docker, it is recommended to mount `/sys/kernel/debug`; otherwise the container cannot access DCU process information.
