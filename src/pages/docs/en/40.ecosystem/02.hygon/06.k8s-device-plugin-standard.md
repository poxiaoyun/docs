## Component description

DCU-Device-Plugin is implemented based on the Kubernetes Device Plugin framework and is responsible for managing DCU/vDCU device resource allocation in the cluster.

It supports multiple working modes and distinguishes application scenarios through the environment variable `RESOURCE_REGISTER_STRATEGY`:

| RESOURCE_REGISTER_STRATEGY | Meaning | Resource type |
| --- | --- | --- |
| dcu | Controls only DCU physical devices | hygon.com/dcu |
| vdcu | Control pre-sharded vDCU only | hygon.com/dcu-share-* |
| mixed (default) | Standard mode: collaborative management and control of physical devices and pre-segmented vDCUs | hygon.com/dcu, hygon.com/dcu-share-* |
| hami | Dynamically split vDCU devices | hygon.com/dcunum |
| mig | MIG virtualization device management and control | hygon.com/dcu-mig-* |

### Standard mode rules

- Supports management and control of physical DCU cards and pre-segmented vDCUs.
- Distinguish physical DCUs and vDCUs of different specifications through resource names, for example:
  - `hygon.com/dcu`: whole card scheduling
  - `hygon.com/dcu-share-40c-30g`: vDCU with 40 CU / 30G video memory
  - `hygon.com/dcu-share-20c-40g`: vDCU with 20 CU / 40G video memory
- DCU cards with pre-segmented vDCUs will not participate in whole-card scheduling; cards with vDCUs must be scheduled through vDCU specification resource names to avoid OOM caused by mixed scheduling.

## Installation and deployment

```bash
kubectl apply -f k8s-dcu-plugin.yaml
```

## Deployment verification

View Pods:

```bash
kubectl get pod -n kube-system -owide | grep dcu-device-plugin
```

View node resource registration:

```bash
kubectl describe node <node-name>
```

Example: `hygon.com/dcu: 2` appears in the Capacity field, indicating that the node has two DCUs.

## Application examples

### Apply for full card DCU

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: dcu-pytorch-demo
spec:
  containers:
    - name: dcu-pytorch-demo
      image: image.sourcefind.cn:5000/dcu/admin/base/pytorch:2.1.0-ubuntu22.04-dtk24.04.2-py3.10
      command: ["/bin/bash", "-c", "--"]
      args: ["sleep infinity & wait"]
      resources:
        limits:
          hygon.com/dcu: 1
```

### Apply for pre-sharded vDCU

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: dcu-share-pytorch-demo
spec:
  containers:
    - name: dcu-share-pytorch-demo
      image: image.sourcefind.cn:5000/dcu/admin/base/pytorch:2.1.0-ubuntu22.04-dtk24.04.2-py3.10
      command: ["/bin/bash", "-c", "--"]
      args: ["sleep infinity & wait"]
      resources:
        limits:
          hygon.com/dcu-share-15c-8g: 1
```

Usage restrictions: Each container can apply for up to 1 vDCU. When applying for more than 1 vDCU, the k8s resource pool will reduce the number of applications and only 1 vDCU will take effect.
