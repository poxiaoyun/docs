## illustrate

MIG mode is used to manage and control divided MIG instances on the DCU. Each MIG instance is exposed to the outside world as an independent schedulable resource.

Using MIG mode requires the host to enable the MIG function and create the required MIG devices. For details, please refer to the "Hygon DCU Multi-Instance User Manual".

:::warning
After the MIG function is enabled, the vDCU virtualization function is unavailable.
:::

## Installation and deployment

```bash
kubectl label nodes <node-name> dcu-mode=mig
kubectl apply -f k8s-dcu-plugin-mig.yaml
```

## Deployment verification

```bash
kubectl get pod -n kube-system -owide | grep dcu-device-plugin-mig
```

View node resource registration:

```bash
kubectl describe node <node-name>
```

Example: `hygon.com/dcu-mig-2g-15gb: 1` appears in the Capacity field, indicating that the node has a MIG device with corresponding specifications.

## Application examples

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: dcu-mig-pytorch-demo
spec:
  containers:
    - name: dcu-mig-pytorch-demo
      image: image.sourcefind.cn:5000/dcu/admin/base/pytorch:2.1.0-ubuntu22.04-dtk24.04.2-py3.10
      command: ["/bin/bash", "-c", "--"]
      args: ["sleep infinity & wait"]
      resources:
        limits:
          hygon.com/dcu-mig-4g-31gb: 1
```
