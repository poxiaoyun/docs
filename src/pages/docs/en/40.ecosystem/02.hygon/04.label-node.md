## Installation and deployment

The k8s cluster deploys DCU-Label-Node in the form of DaemonSet:

```bash
kubectl apply -f k8s-dcu-label-node.yaml
```

## Deployment verification

Check the running status of Pod:

```bash
kubectl get pod -n kube-system -owide | grep dcu-label-node
```

## Node label description

DCU-Label-Node will check the DCU status on the node when running, and create a label to describe the DCU information on the node where DCU is confirmed to be configured.

| Tag | Type | Description |
| --- | --- | --- |
| hygon.com/dcu | string | If the node has a DCU device, the value is `true`; otherwise, labels prefixed with `hygon.com/dcu` will not be automatically generated |
| hygon.com/dcu.name | string | DCU model (example: hygon.com/dcu.name=K100_AI) |
| hygon.com/dcu.cu-count | string | Number of CUs for DCU (Example: hygon.com/dcu.cu-count=120) |
| hygon.com/dcu.vram | string | DCU memory capacity (example: hygon.com/dcu.vram=64G) |

Note: `hygon.com/dcu.name`, `hygon.com/dcu.cu-count`, `hygon.com/dcu.vram` are automatically generated by default, and users can turn them on or off through YAML startup parameters.

## Further confirmation

Users can confirm whether the label is set successfully through the node description information:

```bash
kubectl describe node <node-name>
```
