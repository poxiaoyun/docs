# 节点与 GPU 仪表盘

节点页与 GPU 页用于从“机器视角”观察集群运行状况，是平台运维排查资源瓶颈的主要入口。

## 进入路径

- Boss -> Rune 智算管理 -> 集群管理 -> 选择集群 -> 节点
- Boss -> Rune 智算管理 -> 集群管理 -> 选择集群 -> GPU 仪表盘

## 节点仪表盘

节点仪表盘聚合所有名称中包含 `node` 的监控看板，适合关注：

- 节点在线状态
- CPU 与内存使用率
- 节点压力与异常告警
- 节点级负载分布

## GPU 仪表盘

GPU 仪表盘聚合所有名称中包含 `gpu` 的监控看板，适合关注：

- GPU 利用率
- 显存占用
- 卡级别负载不均衡
- 特定型号设备的热点与空闲情况

## 推荐排查顺序

1. 先看集群总览确认是整体异常还是局部异常。
2. 若怀疑节点故障，进入节点仪表盘确认受影响机器。
3. 若怀疑算力不足或卡分配不均，再进入 GPU 仪表盘。
4. 需要进一步落到对象层时，再结合 Kubernetes 资源浏览或日志页面。
