## 概述
BOSS 负责审核 Rune 推理托管的域名、证书与访问策略，确保对外服务满足安全准则。本章节提供审核流程与常用检查项。

## 前置条件
- 推理服务已在 Rune 中创建并提交审核。
- 管理员具备 `GatewayAuditor` 权限。

## 操作步骤
1. **查看待审核列表**：BOSS → 网关审核，筛选状态 `Pending` 的服务。
2. **检查域名**：确认域名归属、证书有效期与解析状态。
3. **校验策略**：核对限流、IP 白名单、Header 透传等设置。
4. **发布结果**：通过/驳回并附上意见，系统会同步到 Rune 并通知服务负责人。

## 示例
```bash
rune gateway approve \
  --service demo-inference \
  --comment "证书有效至 2026-01-01"
```

## 常见问题
- **Q：审核通过但服务仍不可访问？**
  **A：** 检查域名解析是否生效，必要时在网关层刷新缓存。
- **Q：如何引入第三方 WAF？**
  **A：** 在审核流程中选择“外部 WAF”，并配置回源策略。
