## 概述
BOSS 允许管理员接入多套算力集群，并基于标签与优先级实现跨集群调度。本指南涵盖集群注册、节点巡检与容量监控。

## 前置条件
- 具备访问集群 API 的凭证。
- 已部署 Rune Agent 以同步节点状态。

## 操作步骤
1. **注册集群**：在 BOSS → 集群 → “添加集群”，填写 Kubeconfig 或使用托管集群连接。
2. **节点标记**：结合 `gpu-type`、`zone` 等标签，定义调度策略。
3. **健康巡检**：配置自动巡检计划，定期执行 `rune cluster health`。
4. **容量监控**：启用容量告警并与企业消息通道对接。

## 示例
```bash
rune cluster register \
  --name gpu-prod \
  --kubeconfig kubeconfig-prod.yaml
```

## 常见问题
- **Q：节点状态 Unknown？**
  **A：** 检查 Agent 是否正常运行，必要时重启 DaemonSet 并查看日志。
- **Q：如何隔离不同 GPU 代际？**
  **A：** 使用节点标签 `gpu.generation` 并在模板中指定 `nodeSelector`。
