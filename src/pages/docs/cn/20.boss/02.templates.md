## 概述
系统模板用于向 Rune 用户输出受控的环境配置（如镜像、卷、网络策略）。BOSS 负责模板的版本管理、审批与分发。

## 前置条件
- 已连接企业镜像仓库与配置仓库。
- 模板发布者具备 `TemplateAdmin` 权限。

## 操作步骤
1. **撰写模板**：遵循平台 Schema，定义参数化字段与约束。
2. **提交审核**：在 BOSS 中上传模板包，填写影响范围与回滚方案。
3. **灰度发布**：选择目标租户/项目试运行，收集反馈后全量上线。
4. **版本冻结**：将旧版本标记为 `Deprecated`，提示用户迁移。

## 示例
```bash
rune template bundle \
  --path ./templates/gpu-train \
  --output gpu-train.tgz
```

## 常见问题
- **Q：模板参数如何做合法性校验？**
  **A：** 在 Schema 中定义 `enum`/`pattern`，并在 Rune 端渲染校验信息。
- **Q：能否自动同步 Git 仓库？**
  **A：** 可启用“Git Webhook”将模板仓库与 BOSS 绑定，实现提交即发布。
