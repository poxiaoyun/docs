## 概述
状态监控用于跟踪平台健康度，包含控制面、数据面与作业层指标。本章节提供仪表板入口与告警策略建议。

## 监控面板
- **控制面**：API 请求成功率、延迟、错误码分布。
- **数据面**：队列长度、GPU/CPU 使用率、存储 I/O。
- **作业层**：工作负载状态、失败原因、重试次数。

## 操作步骤
1. 登录 Grafana，选择“Rune Platform Overview”仪表板。
2. 根据业务分组筛选租户或项目。
3. 若发现异常，点击卡片跳转到相关日志面板。

## 告警建议
- API 成功率 < 99% 连续 5 分钟触发 P1 告警。
- GPU 使用率 > 90% 持续 30 分钟提示扩容。
- 作业失败率 > 10% 时通知研发团队。

## 常见问题
- **面板无数据？** 检查监控数据源配置，确保 Prometheus 正常运行。
- **告警过多？** 调整阈值并设置静默策略，避免告警风暴。
