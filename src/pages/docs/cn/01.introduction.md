# Rune Docs

Rune Docs 是一个基于 React、Vite 和 Material UI 构建的现代、高性能文档站点。它具有自动导航生成、国际化支持以及简洁、响应式的设计。

## 特性

- 🚀 **高性能**：基于 Vite 和 React 19 构建。
- 📝 **Markdown 支持**：使用支持 GFM 的标准 Markdown 编写文档。
- 🌍 **国际化**：内置多语言支持（英语和中文）。
- 🎨 **Material Design**：基于 MUI 的美观且易用的 UI 组件。
- 🌙 **深色/浅色模式**：无缝主题切换。
- 📂 **自动生成导航**：导航菜单根据文件结构自动生成。

## 先决条件

- Node.js >= 20

## 安装

**使用 Yarn（推荐）**

```sh
yarn install
```

**使用 Npm**

```sh
npm install
```

## 开发

启动开发服务器：

```sh
yarn dev
# 或
npm run dev
```

在浏览器中打开 [http://localhost:8080](http://localhost:8080) 查看。

## 构建

构建生产版本：

```sh
yarn build
# 或
npm run build
```

## 文档使用

文档内容位于 `src/pages/docs`。站点支持多语言，每种语言有独立的目录（例如 `cn` 代表中文，`en` 代表英语）。

### 目录结构

```text
src/pages/docs/
  ├── cn/                  # 中文文档
  │   ├── 01.introduction.md
  │   ├── 02.development/  # 分组部分
  │   │   ├── index.md     # 文件夹元数据
  │   │   └── ...
  │   └── ...
  └── en/                  # 英文文档
      ├── 01.introduction.md
      └── ...
```

### 添加新页面

1.  **创建文件**：在相应的语言目录中添加 `.md` 文件。
2.  **命名约定**：使用数字前缀控制顺序（例如 `01.getting-started.md`）。数字将从 URL 中剥离。
3.  **Frontmatter**：在文件顶部添加 YAML frontmatter 块以定义页面标题。

```markdown
---
title: 介绍
---

# 欢迎使用 Rune Docs

在此处输入内容...
```

### 创建分组（文件夹）

将相关页面分组到一个部分：

1.  创建一个文件夹（例如 `02.development`）。
2.  在文件夹内，创建一个 `index.md` 文件。
3.  在 `index.md` 中，使用 frontmatter 指定该组的标题。

```markdown
---
title: 开发指南
---
```

_注意：`index.md` 文件仅用于文件夹元数据，本身不会成为可见页面。_

### 国际化 (i18n)

- 将英语文件放在 `src/pages/docs/en/` 中。
- 将中文文件放在 `src/pages/docs/cn/` 中。
- 如果缺少翻译，系统会自动回退到默认语言（中文）。
