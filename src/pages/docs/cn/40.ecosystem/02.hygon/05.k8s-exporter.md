## 监控指标

DCU-Exporter 通过 HTTP 接口暴露 DCU 的监控指标：默认端口 `16080`，默认路径 `/metrics`。

访问示例：`http://<host>:16080/metrics`

## 安装部署

k8s 集群一般以 DaemonSet 形式部署：

```bash
kubectl create -f dcu-exporter-v2.yaml
```

### 端口与指标采集配置

- 如需自定义端口，可修改 `dcu-exporter-v2.yaml` 中 `containerPort`、`prometheus.io/port` 和 `service port`。
- 默认采集全部指标，可用环境变量 `ENABLE_METRICS` 指定需要采集的指标（用英文逗号分隔）。
- 指标采集间隔可用 `PULSE`（秒）设置。
- 日志输出级别/等级/开关通过 `LOG*` 环境变量配置。

## 部署验证

查看 Pod：

```bash
kubectl get pod -n kube-system -owide | grep dcu-exporter
```

通过 curl 查看指标：

```bash
curl localhost:16080/metrics
```

若集群已部署 Prometheus，可在 Prometheus 中查询相关指标验证抓取情况。

## 物理机环境安装部署

除容器化部署外，用户可以通过一键安装方式在物理机安装部署 DCU-Exporter（需保证 DCU 驱动已安装）。

- 从社区下载 `dcu-exporter.run`
- 执行安装：

```bash
bash dcu-exporter.run
```

安装目录：`/opt/dcu-exporter`

启动示例：

```bash
/opt/dcu-exporter/dcu-exporter -h
```
