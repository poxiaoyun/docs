## 安装部署

k8s 集群以 DaemonSet 形式部署 DCU-Label-Node：

```bash
kubectl apply -f k8s-dcu-label-node.yaml
```

## 部署验证

查看 Pod 运行情况：

```bash
kubectl get pod -n kube-system -owide | grep dcu-label-node
```

## 节点标签说明

DCU-Label-Node 运行时会检查节点上 DCU 状态，并在确认配置了 DCU 的节点上创建标签来描述 DCU 信息。

| 标签 | 类型 | 描述 |
| --- | --- | --- |
| hygon.com/dcu | string | 若节点存在 DCU 设备，值为 `true`；否则 `hygon.com/dcu` 为前缀的标签均不自动生成 |
| hygon.com/dcu.name | string | DCU 型号（示例：hygon.com/dcu.name=K100_AI） |
| hygon.com/dcu.cu-count | string | DCU 的 CU 数量（示例：hygon.com/dcu.cu-count=120） |
| hygon.com/dcu.vram | string | DCU 显存容量（示例：hygon.com/dcu.vram=64G） |

备注：`hygon.com/dcu.name`、`hygon.com/dcu.cu-count`、`hygon.com/dcu.vram` 默认自动生成，用户可通过 YAML 启动参数开启或关闭。

## 进一步确认

用户可通过节点描述信息确认标签是否设置成功：

```bash
kubectl describe node <node-name>
```
