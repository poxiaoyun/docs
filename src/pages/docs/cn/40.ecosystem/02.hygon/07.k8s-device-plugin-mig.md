## 说明

MIG 模式用于管控 DCU 上已划分的 MIG 实例，每个 MIG 实例作为独立的可调度资源对外暴露。

使用 MIG 模式需要宿主机开启 MIG 功能并创建所需 MIG 设备，具体可参考《Hygon DCU Multi-Instance 使用手册》。

<div data-alert="warning">

开启 MIG 功能之后，vDCU 虚拟化功能不可用。

</div>

## 安装部署

```bash
kubectl label nodes <node-name> dcu-mode=mig
kubectl apply -f k8s-dcu-plugin-mig.yaml
```

## 部署验证

```bash
kubectl get pod -n kube-system -owide | grep dcu-device-plugin-mig
```

查看节点资源注册：

```bash
kubectl describe node <node-name>
```

示例：Capacity 字段出现 `hygon.com/dcu-mig-2g-15gb: 1` 表示节点存在对应规格的 MIG 设备。

## 应用实例

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: dcu-mig-pytorch-demo
spec:
  containers:
    - name: dcu-mig-pytorch-demo
      image: image.sourcefind.cn:5000/dcu/admin/base/pytorch:2.1.0-ubuntu22.04-dtk24.04.2-py3.10
      command: ["/bin/bash", "-c", "--"]
      args: ["sleep infinity & wait"]
      resources:
        limits:
          hygon.com/dcu-mig-4g-31gb: 1
```
