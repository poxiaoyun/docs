## 组件说明

DCU-Device-Plugin 基于 Kubernetes Device Plugin 框架实现，负责在集群中管理 DCU/vDCU 设备资源分配。

它支持多种工作模式，并通过环境变量 `RESOURCE_REGISTER_STRATEGY` 区分应用场景：

| RESOURCE_REGISTER_STRATEGY | 含义 | 资源类型 |
| --- | --- | --- |
| dcu | 仅管控 DCU 物理设备 | hygon.com/dcu |
| vdcu | 仅管控预切分 vDCU | hygon.com/dcu-share-* |
| mixed（默认） | 标准模式：物理设备与预切分 vDCU 协同管控 | hygon.com/dcu、hygon.com/dcu-share-* |
| hami | 动态切分 vDCU 设备 | hygon.com/dcunum |
| mig | MIG 虚拟化设备管控 | hygon.com/dcu-mig-* |

### 标准模式规则

- 支持物理 DCU 卡与预切分 vDCU 的管控。
- 通过资源名称区分物理 DCU 和不同规格的 vDCU，例如：
  - `hygon.com/dcu`：整卡调度
  - `hygon.com/dcu-share-40c-30g`：40 CU / 30G 显存的 vDCU
  - `hygon.com/dcu-share-20c-40g`：20 CU / 40G 显存的 vDCU
- 存在预切分 vDCU 的 DCU 卡不会参与整卡调度；存在 vDCU 的卡必须通过 vDCU 规格资源名称调度，避免混合调度导致 OOM。

## 安装部署

```bash
kubectl apply -f k8s-dcu-plugin.yaml
```

## 部署验证

查看 Pod：

```bash
kubectl get pod -n kube-system -owide | grep dcu-device-plugin
```

查看节点资源注册：

```bash
kubectl describe node <node-name>
```

示例：Capacity 字段出现 `hygon.com/dcu: 2` 表示节点有两块 DCU。

## 应用实例

### 申请整卡 DCU

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: dcu-pytorch-demo
spec:
  containers:
    - name: dcu-pytorch-demo
      image: image.sourcefind.cn:5000/dcu/admin/base/pytorch:2.1.0-ubuntu22.04-dtk24.04.2-py3.10
      command: ["/bin/bash", "-c", "--"]
      args: ["sleep infinity & wait"]
      resources:
        limits:
          hygon.com/dcu: 1
```

### 申请预切分 vDCU

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: dcu-share-pytorch-demo
spec:
  containers:
    - name: dcu-share-pytorch-demo
      image: image.sourcefind.cn:5000/dcu/admin/base/pytorch:2.1.0-ubuntu22.04-dtk24.04.2-py3.10
      command: ["/bin/bash", "-c", "--"]
      args: ["sleep infinity & wait"]
      resources:
        limits:
          hygon.com/dcu-share-15c-8g: 1
```

使用限制：每个 container 最多可以申请 1 张 vDCU。申请多于 1 张时，k8s 资源池会削减申请数量，仅 1 张 vDCU 生效。
