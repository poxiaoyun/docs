## 适用范围

需要使用以下能力的场景通常要求安装 Ascend Docker Runtime：

- 容器化支持
- 整卡/虚拟化调度
- 断点续训、弹性训练
- 推理卡故障恢复/重调度

如果仅做资源监测，通常可以跳过该组件。

## 前置条件

- 建议使用 root 用户安装
- 确认 `runc` 文件的用户 ID 为 0
- 明确你的运行时场景：Docker / containerd / K8s 集成 containerd

## 确认安装场景

- 查询节点：

```bash
kubectl get node
```

- 查看运行时版本：

```bash
docker --version
containerd --version
```

- K8s 场景可通过节点描述确认运行时：

```bash
kubectl describe node <node-name> | grep -i runtime
```

## 安装（Docker 场景）

在所有计算节点执行：

```bash
cd <path-to-run>
chmod u+x Ascend-docker-runtime_{version}_linux-{arch}.run
./Ascend-docker-runtime_{version}_linux-{arch}.run --check
./Ascend-docker-runtime_{version}_linux-{arch}.run --install
systemctl daemon-reload && systemctl restart docker
```

如需指定安装路径（必须是绝对路径）：

```bash
./Ascend-docker-runtime_{version}_linux-{arch}.run --install --install-path=<abs-path>
```

## 安装（containerd 场景）

```bash
cd <path-to-run>
chmod u+x Ascend-docker-runtime_{version}_linux-{arch}.run
./Ascend-docker-runtime_{version}_linux-{arch}.run --check
./Ascend-docker-runtime_{version}_linux-{arch}.run --install --install-scene=containerd
systemctl daemon-reload && systemctl restart containerd
```

:::warning
Ascend Device Plugin 会在启动时检测 Ascend Docker Runtime 是否存在。通常需要先安装并重启容器引擎，再启动/重启 Device Plugin。
:::

## 常见问题提示

- 如果你的 Docker/Containerd 配置文件路径不是默认位置，安装时可能需要额外指定 `--config-file-path`。
- containerd 场景如安装失败，往往需要检查 `config.toml` 中 runtime 配置与 cgroup 版本匹配（v1/v2），并确保 runtime 路径指向正确的 `ascend-docker-runtime`。
