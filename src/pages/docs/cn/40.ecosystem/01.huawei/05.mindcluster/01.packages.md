## 概述

在手动安装集群调度组件前，通常需要准备对应版本的软件包与镜像构建材料。MindCluster 组件往往以“组件二进制 + Dockerfile/YAML + 配置文件”的形式交付。

本文仅整理交付要点：**下载范围**、**签名校验**、**源码入口**。

## 组件与交付物（常见）

不同版本/形态下内容略有差异，但常见会包含：

- **Ascend Docker Runtime**：运行时增强与挂载能力（含安装 run 包、工具与默认挂载列表）
- **NPU Exporter**：NPU 指标采集（含二进制、Dockerfile、DaemonSet YAML）
- **Ascend Device Plugin**：设备注册与调度对接（含二进制、Dockerfile、适配不同型号/是否 Volcano 的 YAML）
- **Volcano 相关**（可选）：调度器组件与 NPU 插件库
- **NodeD**：节点状态/故障监测上报（含 DaemonSet YAML、配置文件、Dockerfile）

## 下载与版本变量

官方文档中常以占位符表示版本与架构：

- `{version}`：软件版本号（示例：`7.2.RC1`）
- `{arch}`：CPU 架构（示例：`x86_64` / `aarch64`）

建议在交付单中固定记录：组件版本、镜像 tag、运行时类型、K8s 版本范围。

## 软件数字签名验证（建议）

为降低传输/存储过程中被篡改的风险，建议下载软件包的同时下载对应签名文件，并在安装/升级前完成 PGP 校验。

- 校验方法参考官方提供的 OpenPGP 签名验证指南
- 如校验失败，停止使用该包并联系技术支持排查

## 开源组件源码（可选）

如需阅读源码或进行二次开发，可从官方提供的仓库路径获取相应组件源码（以官方列表为准）。常见路径包括：

- Ascend Docker Runtime
- NPU Exporter
- Ascend Device Plugin
- Volcano（Ascend for Volcano）
- Ascend Operator / NodeD / ClusterD / TaskD
