## 适用范围

使用以下能力时通常需要部署 NodeD：

- 整卡/虚拟化调度
- 断点续训、弹性训练
- 推理卡故障恢复/重调度

## 操作步骤（DaemonSet）

### 1）确认镜像存在

```bash
docker images | grep noded
```

### 2）准备 YAML

将软件包解压目录内的 YAML 拷贝到 K8s 管理节点。

### 3）（可选）配置上报间隔

断点续训/弹性训练等场景，可能需要调整节点状态上报间隔（示例）：

```yaml
args: ["/usr/local/bin/noded -logFile=/var/log/mindx-dl/noded/noded.log -logLevel=0 -reportInterval=5"]
```

说明：K8s 默认在一段时间未收到节点响应时会将节点置为 `NotReady`；当 APIServer 压力较大时，可适当增大 `reportInterval` 以减轻压力（需结合稳定性评估）。

### 4）启动

- 不使用 DPC 故障检测时：

```bash
kubectl apply -f noded-v{version}.yaml
```

- 使用 DPC 故障检测且环境满足版本要求时：

```bash
kubectl apply -f noded-dpc-v{version}.yaml
```

### 5）验证

```bash
kubectl get pod -n mindx-dl | grep noded
```

出现 `Running` 通常表示启动成功。
