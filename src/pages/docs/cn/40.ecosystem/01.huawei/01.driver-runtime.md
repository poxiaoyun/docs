## 概述

本页聚焦 **openEuler 场景**下的昇腾 NPU **驱动与固件**安装流程与验证方法，适用于物理机/容器部署（在物理机安装）以及虚拟机场景（物理机装驱动+固件，虚拟机仅装驱动）。

本文以昇腾社区 CANN Community Edition 的安装指南为参考做了“面向交付”的整理，便于在生态文档中快速查阅。

## 前置条件

- 已确认节点 OS 为 openEuler，内核版本与目标驱动版本兼容
- 已准备驱动包与固件包（`.run` 安装包），并能在目标机器上访问到
- 已确认安装/升级场景：首次安装 or 覆盖安装（决定安装顺序）
- 具备 root 权限（安装动作通常需要 root）

:::tip
如果你还需要安装 CANN（ToolKit/Kernel/Runtime）本体，请在驱动/固件完成并验证通过后，再按官方“安装 CANN”章节继续。
:::

## 操作步骤

### 1）确认是否已安装驱动

可先执行以下命令检查当前环境是否已有驱动信息：

```bash
npu-smi info
```

若能返回 NPU/驱动相关信息，说明已安装；否则视为未安装或未加载成功。

### 2）确定安装顺序（首次安装 vs 覆盖安装）

- **首次安装**（设备出厂未装 / 已卸载后重装）：按 **驱动 → 固件** 顺序安装
- **覆盖安装**（已安装且未卸载，重新安装/升级）：按 **固件 → 驱动** 顺序安装

### 3）准备运行用户（默认 HwHiAiUser）

驱动/固件安装时默认运行用户/用户组为 `HwHiAiUser`。若环境中没有该用户/组，建议提前创建：

```bash
groupadd HwHiAiUser
useradd -g HwHiAiUser -d /home/HwHiAiUser -m HwHiAiUser -s /bin/bash
```

如需使用其他运行用户，安装时需要显式指定（见下方“安装驱动”步骤）。

:::warning
如果安装驱动/固件时未使用 `--install-for-all`，则后续 CANN 软件包的运行用户通常需要与驱动/固件运行用户处于同一用户组，否则可能出现权限/访问问题。
:::

### 4）安装依赖（openEuler）

刷新 yum 缓存：

```bash
yum makecache
```

安装编译/内核相关依赖（示例以当前内核为准）：

```bash
yum install -y make dkms gcc kernel-headers-$(uname -r) kernel-devel-$(uname -r)
```

若出现依赖缺失或 dkms 相关报错，通常需要检查软件源可用性或按官方“安装依赖/驱动报错处理”章节补齐依赖。

### 5）安装驱动与固件

将驱动包与固件包上传到目标机器任意目录（示例：`/home`），进入该目录并赋予执行权限：

```bash
chmod +x Ascend-hdk-<chip_type>-npu-driver_<version>_linux-<arch>.run
chmod +x Ascend-hdk-<chip_type>-npu-firmware_<version>.run
```

安装路径通常为 `/usr/local/Ascend`（以实际安装回显为准）。

#### 安装驱动

示例（全量安装 + 对所有用户生效）：

```bash
./Ascend-hdk-<chip_type>-npu-driver_<version>_linux-<arch>.run --full --install-for-all
```

若你没有使用默认运行用户/用户组，可在安装驱动时指定：

```bash
./Ascend-hdk-<chip_type>-npu-driver_<version>_linux-<arch>.run \
  --full \
  --install-username=<username> \
  --install-usergroup=<usergroup>
```

#### 安装固件

```bash
./Ascend-hdk-<chip_type>-npu-firmware_<version>.run --full
```

安装完成后，按安装回显提示决定是否需要重启：

```bash
reboot
```

## 验证方法

重启（或按提示完成加载）后，执行：

```bash
npu-smi info
```

若能返回驱动相关信息，通常说明驱动加载成功。

如你是容器部署场景，建议再补充一次“容器内最小验证”：启动一个具备 NPU 可见性的容器镜像，在容器内执行最小算子/推理样例（以你们平台内置镜像与验证脚本为准）。

## 常见问题

### npu-smi 无输出或看不到设备

- 确认是否按安装回显要求重启
- 确认驱动安装是否成功（安装回显中应出现安装成功的关键提示）
- 检查依赖是否齐全（dkms、kernel headers/devel 等）

### 安装过程中提示缺少 Linux 工具 / dkms / 依赖

- 先检查 yum 源是否可用：`yum makecache`
- 再补齐依赖：`make` / `dkms` / `gcc` / `kernel-headers` / `kernel-devel`

### 覆盖安装顺序不一致导致异常

- 已安装未卸载的覆盖安装，一般按 **固件 → 驱动** 顺序
- 首次安装一般按 **驱动 → 固件** 顺序
