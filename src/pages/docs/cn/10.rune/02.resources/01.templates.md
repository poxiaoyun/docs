## 概述
标准化的镜像与模板可以显著缩短工作负载配置时间。通过集中管理镜像仓库、模板仓库以及版本标签，团队可以保证环境一致并降低安全风险。

## 前置条件
- 已接入企业镜像仓库或使用平台托管仓库。
- 拥有模板发布权限。

## 操作步骤
1. **镜像命名规范**：遵循 `<registry>/<project>/<component>:<semver>`，并使用签名策略。
2. **模板结构**：模板由 `workload.yaml` + `metadata.yaml` 组成，通过 `rune template publish` 发布。
3. **版本冻结**：在创建工作负载时锁定模板版本，避免潜在破坏性升级。
4. **安全扫描**：在 CI 中集成镜像扫描并上传报告至 BOSS。

## 示例
```bash
rune template publish \
  --name transformer-train \
  --version 1.2.0 \
  --file workload.yaml
```

## 常见问题
- **Q：模板更新后旧工作负载会自动升级吗？**
  **A：** 不会。需在控制台手动选择新模板版本或使用 CLI 触发滚动更新。
- **Q：镜像推送失败？**
  **A：** 检查仓库凭证与网络。若使用私有证书，需在节点上配置信任根证书。
