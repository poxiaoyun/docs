# 实例模板与复用

实例模板（Instance Template）用于保存一份可重复使用的实例配置，便于团队在同一工作空间内快速复用推理、微调等工作负载的参数。

## 进入路径

Rune 工作台 -> 资源管理 -> 实例模板

典型路径：`/rune/tenants/:tenant/clusters/:cluster/workspaces/:workspace/instance-templates`

## 模板的来源

当前前端里，模板主要有两种来源：

| 来源 | 说明 |
| --- | --- |
| 从已有实例另存为模板 | 在实例列表或实例详情中执行“保存为模板” |
| 编辑已有模板 | 在模板列表中进入编辑页，更新描述和参数 |

## 模板列表页

模板列表会展示以下核心字段：

| 字段 | 说明 |
| --- | --- |
| 模板名称 | 模板的展示名称 |
| 模板分类 | 当前代码中重点支持 `inference`、`tune` 等分类 |
| 关联产品 | 模板来源产品及版本 |
| 创建者 | 模板的创建人 |
| 创建时间 | 模板创建时间 |

列表中的常见操作：

- 部署：直接从模板创建新实例
- 编辑：修改模板信息和参数
- 删除：移除不再需要的模板

## 从实例保存为模板

实例详情中的“保存为模板”会要求填写：

| 字段 | 说明 |
| --- | --- |
| ID | 模板唯一标识 |
| 名称 | 模板显示名称 |
| 描述 | 用于记录模板用途、场景或约束 |

保存时，系统会记录来源实例的租户、集群和工作空间引用，后续团队成员即可直接复用。

## 从模板部署实例

模板列表中的“部署”操作会弹出表单，要求填写新实例的：

- ID
- 名称
- 描述

提交后，系统会继承模板中的：

- 产品信息
- 分类信息
- values 参数
- 扩展配置

然后跳转到对应的实例列表页。

## 模板编辑方式

模板编辑页当前支持两种参数维护模式：

| 模式 | 说明 |
| --- | --- |
| 表单模式 | 若产品版本提供 JSON Schema，则用动态表单编辑 |
| JSON 模式 | 若 Schema 不可用，或用户主动切换，则直接编辑 JSON |

> 💡 提示: 当产品版本对应的 Schema 获取失败时，前端会自动退回到 JSON 编辑模式。

## 适用场景

- 团队沉淀一套标准的推理或微调参数配置。
- 新成员快速复用已有最佳实践。
- 将一次性调试成功的实例转成模板，减少重复配置。
