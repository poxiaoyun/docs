# 算力规格

算力规格（Flavor）定义了一组可直接选择的资源组合，例如 CPU、内存和 GPU 型号。用户在创建推理、微调、开发环境和应用时，实际选择的就是某个规格。

## 进入路径

Rune 工作台 -> 资源管理 -> 算力规格

典型路径：`/rune/tenants/:tenant/flavors`

## 当前前端能力

根据 `xiaoshi-rune-console` 前端实现，规格页具备以下能力：

| 能力 | 说明 |
| --- | --- |
| 按区域查看 | 需要结合当前选中的 region/cluster 查询 |
| 按规格维度筛选 | 使用 `FlavorFilterBar` 按类型、型号等条件筛选 |
| 展示资源组合 | 通过 `FlavorResources` 展示 CPU、内存、加速器等信息 |
| 展示规格属性 | 包括名称、类型、型号和详细资源配置 |

## 列表字段

| 字段 | 说明 |
| --- | --- |
| 名称 | 规格名称，通常是团队约定的标准命名 |
| 类型 | 如 CPU、GPU 或其他资源类型 |
| 型号 | 例如某种 GPU/NPU 型号 |
| 规格明细 | 该规格对应的 CPU、内存、加速器数量等组合 |

## 规格与配额的关系

规格并不等于配额，它们分别回答两个问题：

| 概念 | 回答的问题 |
| --- | --- |
| 配额 | 你最多能用多少资源 |
| 规格 | 你每次部署时可以选什么组合 |

只有同时满足下面两个条件，某个规格才真正可用：

1. 当前集群存在这个规格。
2. 当前租户或工作空间在该规格对应资源上还有可用配额。

## 使用建议

- 推理服务优先选择稳定、可复用的标准规格。
- 微调和实验任务建议按模型规模预留更高显存。
- 同一团队尽量减少自定义零散规格，便于运维和成本统计。

## 常见问题

### 为什么我能看到规格，但部署时不能选？

常见原因包括：

- 当前工作空间配额不足
- 目标集群资源已耗尽
- 某些模板本身限制了可选规格范围

### 为什么不同集群里看到的规格不一样？

规格是集群相关资源，不同集群的硬件类型、资源池和管理员策略都可能不同。
