## 概述
配额策略确保资源在多团队间公平分配。Rune 支持静态配额与按需借用模式，并可结合成本中心导出报告。

## 前置条件
- 已在 BOSS 中为租户分配基础配额。
- 拥有 Workspace 管理权限。

## 操作步骤
1. **查看配额**：`设置 → 配额` 页面列出各资源的上限与已使用值。
2. **申请扩容**：通过“申请配额”提交理由，BOSS 审核后实时生效。
3. **配额策略**：配置 GPU 借用比例、抢占式策略及按队列的优先级。
4. **成本导出**：启用成本分析并下载 CSV，便于对账。

## 示例
```bash
rune quota apply \
  --gpu 20 \
  --memory 256Gi \
  --reason "Q1 推理高峰"
```

## 常见问题
- **Q：配额申请 Pending？**
  **A：** 检查审批流程是否已通知到 BOSS 管理员，可在飞书群@运营同学。
- **Q：如何限制单任务 GPU？**
  **A：** 在模板中设置 `resources.limits`，并启用队列级别的 `max-gpu-per-job` 策略。
