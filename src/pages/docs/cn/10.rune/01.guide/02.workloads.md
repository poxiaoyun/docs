## 概述

工作负载是 Rune 中调度 GPU/CPU、挂载镜像与存储卷的最小单元。本章节示范如何创建训练与推理工作负载，并配置日志、监控与自动扩缩容策略。

## 前置条件

- 已完成[环境准备](./01.prerequisites)。
- 拥有可用的镜像与运行模板。
- 已申请所需配额（GPU、存储卷等）。

## 操作步骤

1. **选择模板**：在“工作负载”面板点击“新建”，选择官方模板或自定义 YAML。
2. **配置容器**：填写镜像地址、启动命令，勾选所需的 GPU/CPU 数量以及内存。
3. **挂载存储**：在“卷”选项卡绑定 PVC 或临时卷，设置只读/读写权限。
4. **注入环境变量**：通过 Secret/ConfigMap 管理敏感信息。
5. **设置扩缩容**：在“自动扩缩容”中启用 HPA，并设置基于 GPU 利用率的阈值。
6. **发布**：确认配额后点击“创建”，等待状态变为 `Running`。

## 示例

```yaml
apiVersion: runedocs/v1
kind: Workload
metadata:
  name: demo-inference
spec:
  template:
    containers:
      - image: registry.example.com/rune/inference:latest
        resources:
          limits:
            nvidia.com/gpu: 1
            cpu: "4"
            memory: 8Gi
```

## 常见问题

- **Q：镜像拉取失败？**
  **A：** 确认镜像仓库凭证配置正确，并在集群节点上测试 `crictl pull`。
- **Q：扩缩容不生效？**
  **A：** 检查监控指标是否上报，若使用自定义指标需在 Prometheus 中注册同名指标。
