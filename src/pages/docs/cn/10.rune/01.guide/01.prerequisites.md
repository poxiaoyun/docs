## 概述

在创建任何 Rune 工作负载之前，需要完成工作区权限、CLI 环境以及访问密钥的初始化。本指南确保团队遵循统一的安全基线，并预置常用模板与网络策略。

## 前置条件

- 已开通 Rune 租户与至少一个项目空间。
- 具备管理访问密钥的权限。
- 工作站安装 Node.js 20+ 与 Yarn 1.22.22。

## 操作步骤

1. **激活账户**：管理员在 BOSS 中创建租户，并为成员分配“Workspace-Admin”角色。
2. **安装 CLI**：

```bash
curl -fsSL https://rune.example.com/cli/install.sh | bash
rune --version
```

3. **配置访问密钥**：在 Rune 控制台 -> 个人设置 -> 访问凭证创建 `AK/SK`，并写入 `~/.rune/credentials`。
4. **同步模板**：执行 `rune template pull --bundle default` 下载官方模板。

## 示例

```bash
rune auth login --endpoint https://console.rune.example.com
rune profile use <workspace>
rune template ls
```

## 常见问题

- **Q：CLI 登录失败并提示证书错误？**
  **A：** 确认本地系统时间正确，并在 `rune auth login` 中添加 `--skip-tls-verify=false` 以强制校验证书。
- **Q：访问密钥泄露如何处理？**
  **A：** 立即在控制台删除旧密钥并生成新密钥，随后触发审计日志导出。
