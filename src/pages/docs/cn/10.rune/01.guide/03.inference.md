## 概述
推理托管用于将模型以 API 形式对外提供服务，支持灰度发布、限流与观测。本指南涵盖服务创建、流量治理与指标采集，帮助团队安全上线模型能力。

## 前置条件
- 已有可用的推理工作负载，且镜像暴露 HTTP/GRPC 端口。
- 拥有网关访问权限。
- 已准备 TLS 证书或使用平台签发证书。

## 操作步骤
1. **创建服务**：在“推理托管”中选择工作负载，填写服务名称、协议与端口映射。
2. **配置网关**：
   - 绑定域名或使用平台分配的二级域名。
   - 选择公网/内网出口，并上传证书（若启用 HTTPS）。
3. **流量策略**：
   - 设置限流（QPS/并发上限）。
   - 开启熔断与重试策略。
4. **灰度发布**：创建新版本，使用权重或 Header 方式分流。
5. **观测**：启用请求日志与指标上报，将链路数据同步至监控面板。

## 示例
```bash
rune inference create \
  --workload demo-inference \
  --ingress public \
  --domain api.demo.example.com \
  --port 9000:8080
```

## 常见问题
- **Q：域名解析失败？**
  **A：** 确认 DNS 已将域名指向平台网关 IP，必要时提交云解析工单。
- **Q：灰度流量未按权重生效？**
  **A：** 检查是否对接了流量网关缓存，可通过调试 Header 强制命中目标版本。
