## 概述
通过标准化的版本发布流程，团队可以记录每次训练产物的来源、数据集与评测指标，同时为社区提供可复现的模型资产。

## 前置条件
- 模型训练产物已上传至对象存储或模型仓库。
- 具备 `ModelPublisher` 权限。

## 操作步骤
1. **创建版本**：在“模型版本”列表点击“发布”，填写版本号、变更说明与可见范围。
2. **关联数据集**：选择本次训练使用的数据集版本，实现血缘追踪。
3. **上传产物**：提供模型权重、配置文件及推理脚本。
4. **提交审核**：若开启审核流程，版本会进入 `Reviewing` 状态。

## 示例
```bash
rune model publish \
  --model llama2-7b \
  --version 1.1.0 \
  --artifact s3://models/llama2-7b-1.1.0
```

## 常见问题
- **Q：版本号如何命名？**
  **A：** 建议使用 `主版本.次版本.修订` 语义化命名，便于自动比对。
- **Q：可以设置付费可见性吗？**
  **A：** 企业版支持为模型版本附加订阅策略，请联系商务开通。
