## 概述
高质量的数据治理流程可确保数据来源合法、安全可控。本章节提供从入库、脱敏到发布的全流程指引。

## 前置条件
- 数据已完成版权与合规评估。
- 具备 `DatasetMaintainer` 权限。

## 操作步骤
1. **入库检查**：上传数据基本信息，包含来源、License 与体量。
2. **脱敏处理**：使用官方脚本执行敏感字段脱敏，记录处理结果。
3. **版本冻结**：创建版本标签并锁定哈希，防止后续修改。
4. **权限控制**：将数据集设为公开/私有，或仅特定组织可见。

## 示例
```bash
rune dataset publish \
  --name fintech-train \
  --version 2.0.0 \
  --manifest manifest.yaml
```

## 常见问题
- **Q：如何同步大体量数据？**
  **A：** 建议使用对象存储直传或离线导入工具，避免浏览器上传超时。
- **Q：版本发布后还能修改文件吗？**
  **A：** 不可修改原文件，可创建基于该版本的新版本并记录变更。
