## CLI 工具概述

`moha` 是 Moha SDK 提供的命令行工具，让你无需编写代码即可完成仓库操作。

### 命令总览

| 命令 | 说明 |
|------|------|
| `moha login` | 登录并保存 Token |
| `moha logout` | 退出登录 |
| `moha whoami` | 查看当前登录状态 |
| `moha upload` | 上传文件夹到仓库 |
| `moha upload-file` | 上传单个文件到仓库 |
| `moha download` | 下载整个仓库 |
| `moha download-file` | 下载单个文件 |
| `moha repo-list` | 列出仓库 |
| `moha repo-create` | 创建仓库 |
| `moha repo-update` | 更新仓库信息 |
| `moha repo-delete` | 删除仓库 |
| `moha repo-info` | 查看仓库详情 |
| `moha branch-create` | 创建分支 |
| `moha branch-delete` | 删除分支 |
| `moha branch-list` | 列出分支 |
| `moha space-deploy` | 发布 Space |
| `moha space-redeploy` | 重新发布 Space |
| `moha space-status` | 查看 Space 状态 |
| `moha ai` | AI 智能助手 |
| `moha ai-config` | 配置 AI 助手 |

### 快速帮助

```bash
# 查看所有命令
moha --help

# 查看特定命令帮助
moha upload --help
moha download --help
```

### 通用选项

所有命令都支持以下认证选项：

| 选项 | 说明 |
|------|------|
| `--base-url` | Hub API 地址（或设置 `MOHA_ENDPOINT` 环境变量） |
| `--token` | 认证令牌（或设置 `MOHA_TOKEN` 环境变量） |
| `--username` | 用户名（或设置 `MOHA_USERNAME` 环境变量） |
| `--password` | 密码（或设置 `MOHA_PASSWORD` 环境变量） |

## 认证命令

### `moha login`

登录并保存 Token 到本地。

**用法**：
```bash
moha login [OPTIONS]
```

**选项**：
| 选项 | 说明 |
|------|------|
| `--username, -u` | 用户名（不指定时交互输入） |
| `--password, -p` | 密码（不指定时交互输入） |
| `--base-url` | Hub API 地址 |
| `--token-dir` | Token 存储目录（默认 `~/.moha`） |

**示例**：

```bash
# 交互式登录
moha login

# 指定用户名和密码
moha login --username alice --password secret

# 指定 Hub 地址
moha login \
  --username alice \
  --password secret \
  --base-url https://moha.your-domain
```

### `moha logout`

退出登录并删除本地 Token。

```bash
moha logout
```

### `moha whoami`

查看当前登录状态。

```bash
moha whoami
```

## 上传命令

### `moha upload` - 上传文件夹

上传本地文件夹到仓库。

**用法**：
```bash
moha upload <folder> <repo_id> [OPTIONS]
```

**参数**：
| 参数 | 说明 | 例子 |
|------|------|------|
| `<folder>` | 本地文件夹路径 | `./my_model` |
| `<repo_id>` | 目标仓库 ID（格式: 组织/仓库名） | `my-org/my-model` |

**选项**：
| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--repo-type, -t` | 仓库类型 (`models`/`datasets`/`spaces`) | `models` |
| `--revision, -r` | 上传到的分支 | `main` |
| `--message, -m` | 提交信息 | 无 |
| `--ignore, -i` | 忽略匹配的文件（glob 模式，可重复） | 无 |
| `--encrypt, -e` | 启用加密 | 禁用 |
| `--encryption-password` | 加密密码（或设置 `MOHA_ENCRYPTION_PASSWORD`） | 无 |
| `--algorithm, -a` | 加密算法（`AES` / `SM4`） | `AES` |
| `--temp-dir` | 加密临时目录 | 无 |

**示例**：

```bash
# 基础上传
moha upload ./my_model my-org/my-model

# 指定用户名密码
moha upload ./my_model my-org/my-model \
  --username alice \
  --password secret

# 忽略临时文件
moha upload ./my_model my-org/my-model \
  --ignore "*.pyc" \
  --ignore "__pycache__" \
  --ignore "*.tmp"

# 上传数据集
moha upload ./my_dataset my-org/my-dataset \
  --repo-type datasets

# 上传到指定分支
moha upload ./my_model my-org/my-model \
  --revision dev

# 启用 AES 加密上传
moha upload ./my_model my-org/my-model \
  --encrypt \
  --encryption-password secret123

# 使用 SM4 国密算法加密
moha upload ./my_model my-org/my-model \
  --encrypt \
  --encryption-password secret123 \
  --algorithm SM4

# 自定义提交信息
moha upload ./my_model my-org/my-model \
  --message "Upload weights for fine-tuning v2"
```

### `moha upload-file` - 上传单个文件

上传单个文件到仓库。

**用法**：
```bash
moha upload-file <file> <repo_id> [OPTIONS]
```

**参数**：
| 参数 | 说明 | 例子 |
|------|------|------|
| `<file>` | 本地文件路径 | `./config.json` |
| `<repo_id>` | 目标仓库 ID | `my-org/my-model` |

**选项**：
| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--path-in-repo, -p` | 文件在仓库中的路径（默认取文件名） | 文件名 |
| `--repo-type, -t` | 仓库类型 | `models` |
| `--revision, -r` | 分支 | `main` |
| `--message, -m` | 提交信息 | 无 |
| `--encrypt, -e` | 启用加密 | 禁用 |
| `--encryption-password` | 加密密码 | 无 |
| `--algorithm, -a` | 加密算法 | `AES` |

**示例**：

```bash
# 上传单个文件
moha upload-file ./config.json my-org/my-model

# 指定仓库内路径
moha upload-file ./model.safetensors my-org/my-model \
  --path-in-repo "weights/model.safetensors"

# 上传并加密
moha upload-file ./model.bin my-org/my-model \
  --encrypt \
  --encryption-password secret123

# 自定义提交信息
moha upload-file ./config.json my-org/my-model \
  --message "Update config for v2.0"
```

## 下载命令

### `moha download` - 下载整个仓库

下载仓库中的所有文件。

**用法**：
```bash
moha download <repo_id> [OPTIONS]
```

**参数**：
| 参数 | 说明 | 例子 |
|------|------|------|
| `<repo_id>` | 仓库 ID | `ai-lab/qwen2-7b` |

**选项**：
| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--local-dir, -o` | 下载到的本地目录 | 当前目录 |
| `--repo-type, -t` | 仓库类型 (`models`/`datasets`/`spaces`) | `models` |
| `--revision, -r` | 分支或标签 | 无 |
| `--include` | 只下载匹配的文件（glob 模式，可重复） | 无限制 |
| `--ignore, -i` | 忽略匹配的文件（glob 模式，可重复） | 无 |
| `--quiet, -q` | 禁用进度条 | 显示进度 |

**示例**：

```bash
# 基础下载
moha download ai-lab/qwen2-7b

# 下载到指定目录
moha download ai-lab/qwen2-7b --local-dir ./my_models

# 只下载权重文件
moha download ai-lab/qwen2-7b \
  --include "*.safetensors"

# 忽略大文件
moha download ai-lab/qwen2-7b \
  --ignore "*.bin"

# 下载数据集
moha download ai-lab/my-dataset \
  --repo-type datasets \
  --local-dir ./datasets

# 下载特定版本
moha download ai-lab/qwen2-7b \
  --revision v1.0 \
  --local-dir ./qwen2-v1

# 静默下载
moha download ai-lab/qwen2-7b --quiet

# 从私有仓库下载
moha download my-org/private-model \
  --username alice \
  --password secret
```

### `moha download-file` - 下载单个文件

只下载仓库中的特定文件。

**用法**：
```bash
moha download-file <repo_id> <filename> [OPTIONS]
```

**参数**：
| 参数 | 说明 | 例子 |
|------|------|------|
| `<repo_id>` | 仓库 ID | `ai-lab/qwen2-7b` |
| `<filename>` | 仓库内的文件路径 | `config.json` |

**选项**：
| 选项 | 说明 |
|------|------|
| `--local-dir, -o` | 下载到的本地目录 |
| `--repo-type, -t` | 仓库类型 |
| `--revision, -r` | 分支或标签 |
| `--quiet, -q` | 禁用进度条 |

**示例**：

```bash
# 下载单个文件
moha download-file ai-lab/qwen2-7b config.json

# 下载到指定目录
moha download-file ai-lab/qwen2-7b config.json \
  --local-dir ./configs

# 下载特定版本的文件
moha download-file ai-lab/qwen2-7b config.json \
  --revision v1.0
```

## 仓库管理命令

### `moha repo-list` - 列出仓库

**用法**：
```bash
moha repo-list [OPTIONS]
```

**选项**：
| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--repo-type, -t` | 仓库类型 (`models`/`datasets`/`spaces`) | `models` |
| `--scope, -s` | 查询范围（`public`/`create`/`favorite`/`organization`） | `public` |
| `--organization, -o` | 组织名称（scope=organization 时必需） | 无 |
| `--mine, -m` | 仅显示我创建的仓库（仅 scope=organization 时有效） | 否 |

**示例**：

```bash
# 列出公开模型
moha repo-list

# 列出公开数据集
moha repo-list --repo-type datasets

# 列出我创建的模型
moha repo-list --scope create

# 列出我收藏的模型
moha repo-list --scope favorite

# 列出组织内的模型
moha repo-list --scope organization --organization my-org

# 列出组织内我创建的模型
moha repo-list --scope organization --organization my-org --mine
```

### `moha repo-create` - 创建仓库

**用法**：
```bash
moha repo-create <repo_id> [OPTIONS]
```

**选项**：
| 选项 | 说明 | 默认值 |
|------|------|--------|
| `--repo-type, -t` | 仓库类型 | `models` |
| `--description, -d` | 仓库描述 | 无 |
| `--visibility, -v` | 可见性（`public`/`internal`/`private`） | `internal` |
| `--license` | 许可证（可重复） | 无 |
| `--tasks` | 任务类型（可重复） | 无 |
| `--languages` | 语言（可重复） | 无 |
| `--tags` | 标签（可重复） | 无 |
| `--frameworks` | 框架（可重复） | 无 |
| `--base-model` | 基础模型（可重复，格式 `组织/模型名`） | 无 |
| `--relationship` | 与基础模型关系（`adapter`/`finetune`/`quantized`/`merge`/`repackage`） | 无 |

**示例**：

```bash
# 创建模型仓库
moha repo-create my-org/my-model

# 创建数据集仓库
moha repo-create my-org/my-dataset --repo-type datasets

# 创建含描述的公开仓库
moha repo-create my-org/my-model \
  --description "Fine-tuned Qwen2 model" \
  --visibility public

# 创建包含元数据的仓库
moha repo-create my-org/my-model \
  --license apache-2.0 \
  --tasks text-generation \
  --languages zh \
  --frameworks pytorch \
  --base-model ai-lab/qwen2-7b \
  --relationship finetune
```

### `moha repo-update` - 更新仓库

更新仓库描述、可见性、元数据等信息。

```bash
moha repo-update my-org/my-model \
  --description "Updated description" \
  --visibility public
```

### `moha repo-delete` - 删除仓库

```bash
# 删除仓库（需确认）
moha repo-delete my-org/old-model

# 跳过确认
moha repo-delete my-org/old-model --yes
```

### `moha repo-info` - 查看仓库详情

```bash
moha repo-info my-org/my-model
moha repo-info my-org/my-dataset --repo-type datasets
```

## 分支管理命令

## AI 智能助手

### `moha ai` - 启动 AI 助手

使用自然语言管理仓库。

```bash
# 交互模式
moha ai

# 非交互模式（单条指令）
moha ai --prompt "列出所有模型"

# 指定模型
moha ai --model gpt-4
```

**选项**：
| 选项 | 说明 |
|------|------|
| `--prompt, -p` | 直接执行单条指令 |
| `--api-base` | AI API 地址（或设置 `MOHA_AI_API_BASE`） |
| `--api-key` | AI API 密钥（或设置 `MOHA_AI_API_KEY`） |
| `--model, -m` | 模型名称（或设置 `MOHA_AI_MODEL`） |
| `--no-stream` | 禁用流式输出 |

### `moha ai-config` - 配置 AI 助手

```bash
# 配置 API
moha ai-config --api-base https://api.example.com --api-key sk_xxx --model gpt-4

# 查看当前配置
moha ai-config --show
```

## 故障排除

### 问题 1：认证失败

```bash
# 重新登录
moha login --username alice --password secret

# 或明确指定 Token
moha upload ./my_model my-org/my-model \
  --token sk_xxxxxxxxxxxx
```

### 问题 2：大文件上传失败

```bash
# SDK 自动处理大文件分块上传
# 如果中途中断，重新运行命令会自动续传
moha upload ./my_model my-org/my-model
```

### 问题 3：文件过滤不生效

```bash
# 注意引号防止 shell 扩展
# upload 使用 --ignore 过滤
moha upload ./my_model my-org/my-model \
  --ignore "*.pyc" \
  --ignore "__pycache__"

# download 使用 --include 选择文件，--ignore 排除文件
moha download ai-lab/qwen2-7b \
  --include "*.safetensors" \
  --ignore "*.bin"
```

## 最佳实践

### 1. 安全规范

```bash
# ✗ 不要在命令行暴露密码
moha upload ./model org/repo --password secret

# ✓ 使用环境变量或提前 login
export MOHA_TOKEN=sk_xxxxxxxxxxxx
moha upload ./model org/repo
```

### 2. 文件选择

```bash
# 忽略不必要文件
moha upload ./my_model org/repo \
  --ignore "*.pyc" \
  --ignore "__pycache__" \
  --ignore ".git"
```

### 3. 版本管理

```bash
# 上传时指定清晰的提交信息
moha upload ./my_model org/repo \
  --message "Release v1.0: Added LoRA support"
```
