## 认证概述

Moha SDK 支持多种认证方式，按推荐度排序：

| 方式 | 环境 | 推荐度 | 说明 |
|------|------|--------|------|
| **Token** | 环境变量 / 代码 | ⭐⭐⭐⭐⭐ | 最安全、最便捷 |
| **用户名/密码** | 环境变量 / 交互式 | ⭐⭐⭐⭐ | 灵活但需保存密码 |
| **本地存储** | `~/.moha/token.json` | ⭐⭐⭐⭐⭐ | `login` 命令自动管理 |

## 方式 1：Token 认证（推荐）

### 获取 Token

#### 方式 A：在 Console 中生成

1. 登录 Console → **个人中心** → **API Key**
2. 点击 **新增 Key**
3. 输入 Key 名称，选择过期时间
4. 保存显示的 Token（仅此时可见）

#### 方式 B：使用 `login` 命令

```bash
moha login --username your_username --password your_password

# 成功后 Token 自动保存到 ~/.moha/token.json
```

#### 方式 C：编程方式生成（高级）

```python
from xiaoshiai_hub import login

# 使用用户名密码登录
token, token_path = login(
    username="your_username",
    password="your_password"
)
print(f"✓ Token 已保存到: {token_path}")

# 登录后，Token 自动保存
# 下次使用会自动加载
```

### 使用 Token 进行认证

#### 方式 1：环境变量（推荐）

```bash
export MOHA_TOKEN=sk_xxxxxxxxxxxx

# 或在 .env 文件中
echo "MOHA_TOKEN=sk_xxxxxxxxxxxx" > .env
```

#### 方式 2：代码中传参

```python
from xiaoshiai_hub import HubClient

client = HubClient(
    base_url="https://moha.your-domain",
    token="sk_xxxxxxxxxxxx"
)

# 或者使用下载函数
from xiaoshiai_hub import moha_hub_download

model = moha_hub_download(
    repo_id="ai-lab/qwen2-7b",
    filename="config.json",
    token="sk_xxxxxxxxxxxx"
)
```

#### 方式 3：自动加载已保存的 Token

```python
from xiaoshiai_hub import load_token, HubClient

# 加载之前保存的 token
token, base_url, username = load_token()

client = HubClient(token=token)
print(f"✓ 已认证为: {username}")
```

## 方式 2：用户名/密码认证

### 场景 1：环境变量

```bash
export MOHA_USERNAME=your_username
export MOHA_PASSWORD=your_password
```

### 场景 2：代码中传参

```python
from xiaoshiai_hub import HubClient

client = HubClient(
    base_url="https://moha.your-domain",
    username="your_username",
    password="your_password"
)
```

### 场景 3：交互式输入（不推荐在生产环境）

```python
from xiaoshiai_hub import HubClient
import getpass

username = input("用户名: ")
password = getpass.getpass("密码: ")

client = HubClient(username=username, password=password)
```

### ⚠️ 安全警告

**不要在代码中硬编码密码！** 推荐做法：

```python
import os
from xiaoshiai_hub import HubClient

# 从环境变量读取
username = os.environ.get("MOHA_USERNAME")
password = os.environ.get("MOHA_PASSWORD")

client = HubClient(username=username, password=password)
```

## Token 持久化与管理

### 自动保存 Token

使用 `login()` 命令时，Token 会自动保存到 `~/.moha/token.json`：

```python
from xiaoshiai_hub import login

token, token_path = login(
    username="your_username",
    password="your_password"
)
# Token 已保存到 ~/.moha/token.json
```

### 手动保存 Token

```python
from xiaoshiai_hub import save_token

# 保存 Token 到默认位置 (~/.moha/token.json)
save_token(
    token="sk_xxxxxxxxxxxx",
    username="your_username",
    base_url="https://moha.your-domain"
)
```

### 加载已保存的 Token

```python
from xiaoshiai_hub import load_token

token, base_url, username = load_token()
print(f"已加载: {username} @ {base_url}")
```

### 删除已保存的 Token

```python
from xiaoshiai_hub import delete_token

delete_token()
print("Token 已删除")
```

### 自定义 Token 存储位置

默认 Token 存储在 `~/.moha/token.json`，可以自定义：

```python
from xiaoshiai_hub import save_token, load_token

custom_token_dir = "/path/to/custom/dir"

# 保存到自定义目录
save_token(
    token="sk_xxxxxxxxxxxx",
    token_dir=custom_token_dir
)

# 从自定义目录加载
token, base_url, username = load_token(token_dir=custom_token_dir)
```

## 认证优先级

SDK 按以下优先级查找认证信息：

```
优先级顺序 (从高到低):
1. 代码中明确传参 (token / username / password)
2. 环境变量 (MOHA_TOKEN / MOHA_USERNAME / MOHA_PASSWORD)
3. 本地保存的 Token (~/.moha/token.json)
4. 无认证 → 使用公开内容
```

**优先级示例**：

```python
from xiaoshiai_hub import HubClient

# 场景 1：明确传参 → 使用传参
client = HubClient(token="token_from_code")
# ✓ 使用 "token_from_code"，忽略环境变量和本地保存

# 场景 2：无明确传参，有环境变量 → 使用环境变量
# export MOHA_TOKEN=token_from_env
client = HubClient()
# ✓ 使用 "token_from_env"，忽略本地保存

# 场景 3：无明确传参，无环境变量，有本地保存 → 使用本地保存
client = HubClient()
# ✓ 自动加载 ~/.moha/token.json
```

## 常见认证场景

### 场景 1：本地开发

使用 `login` 命令一次性登录，后续自动使用保存的 Token：

```bash
moha login --username your_username --password your_password
# Token 已保存到 ~/.moha/token.json
```

然后在代码中自动加载：

```python
from xiaoshiai_hub import HubClient

client = HubClient()  # 自动加载已保存的 Token
```

### 场景 2：CI/CD 流水线

设置环境变量，然后在脚本中使用：

```bash
#!/bin/bash
export MOHA_ENDPOINT=https://moha.your-domain
export MOHA_TOKEN=${MOHA_SDK_TOKEN}  # 从 CI 系统的 secrets 获取

python train.py
```

在 Python 脚本中：

```python
from xiaoshiai_hub import HubClient

client = HubClient()  # 自动读取环境变量
```

### 场景 3：Docker 容器

在 Dockerfile 中设置环境变量：

```dockerfile
FROM python:3.9

# 安装 SDK
RUN pip install xiaoshiai-hub

# 设置认证信息（从 build args 传入）
ARG MOHA_TOKEN
ENV MOHA_TOKEN=${MOHA_TOKEN}
ENV MOHA_ENDPOINT=https://moha.your-domain

COPY train.py .
CMD ["python", "train.py"]
```

构建和运行：

```bash
docker build --build-arg MOHA_TOKEN=sk_xxxxxxxxxxxx -t ml-app .
docker run ml-app
```

### 场景 4：用户账户和权限

检查当前认证的用户信息：

```python
from xiaoshiai_hub import load_token

token, base_url, username = load_token()
print(f"当前用户: {username}")
print(f"Hub 地址: {base_url}")
```

## 调试认证问题

### 检查认证状态

```python
from xiaoshiai_hub import HubClient, load_token
import os

# 打印认证信息（不显示完整 token）
print("认证调试信息：")
print(f"1. 环境变量 MOHA_TOKEN: {bool(os.environ.get('MOHA_TOKEN'))}")
print(f"2. 环境变量 MOHA_USERNAME: {os.environ.get('MOHA_USERNAME', '未设置')}")
print(f"3. 本地保存 Token: ", end="")

try:
    token, base_url, username = load_token()
    print(f"已保存 (用户: {username})")
except:
    print("未保存")

# 测试连接
try:
    client = HubClient()
    models = client.list_repositories("models")
    print(f"✓ 认证成功")
except Exception as e:
    print(f"✗ 认证失败: {e}")
```

### 常见错误及解决

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `AuthenticationError: Authentication failed` | Token 无效或过期 | 重新生成 Token，检查 Token 是否有效期已过 |
| `RepositoryNotFoundError` | 仓库不存在或无访问权限 | 检查仓库 ID，确认有访问权限 |
| `FileNotFoundError` | 文件不存在 | 检查文件路径是否正确 |
